文件

Document是由JSON对象表示的一组键和值对。通过JSON MySQL数据类型，内部使用MySQL二进制JSON对象表示文档。字段的值可以包含其他文档，数组和文档列表。

采集

Collection是一个容器，可以用来将文档存储在MySQL数据库中。

CRUD操作

创建，读取，更新和删除（CRUD）操作是可以在数据库集合或表上执行的四个基本操作。就MySQL而言，这意味着：

  • 创建一个新条目（插入或添加）

  • 阅读条目（查询）

  • 更新条目

  • 删除条目

X插件

使用X协议启用通信的MySQL服务器插件。支持实现X DevAPI的客户端，并使您能够将MySQL用作文档存储。

X协议

与运行X插件的MySQL服务器通信的协议。X协议支持CRUD和SQL操作，通过SASL进行身份验证，允许流式传输（流水线）命令，并且可在协议和消息层上扩展。

19.3将MySQL设置为文档存储

19.3.1安装MySQL Shell
19.3.2启动MySQL Shell

本节介绍如何安装X插件以使MySQL服务器能够使用X协议并使用MySQL Shell作为服务器的客户端。

使用MySQL作为文档存储的先决条件是实现X协议的X插件。如果没有运行X插件，X协议客户端无法连接到服务器。X插件随附MySQL（5.7.12或更高版本） - 安装它不涉及单独的下载。

将MySQL设置为文档存储遵循以下列出的步骤：

  1. 安装或升级到MySQL 5.7.12或更高版本。

    当安装或升级完成后，启动服务器。有关服务器启动说明，请参见第2.10.2节“启动服务器”。

    注意

    MySQL安装程序使您可以同时执行此操作和下一步（安装X插件），以便在Microsoft Windows上进行全新安装。在“ 插件和扩展”屏幕中，选中“ 启用X协议/ MySQL作为文档存储”复选框。安装完成后，确认X插件已安装。

  2. 安装X插件。只要该帐户具有INSERT该mysql.plugin表的特权，就可以使用非root帐户安装该插件。

    在重新配置服务器之前，始终保存您现有的配置设置。

    要安装内置的X插件，请执行以下操作之一：

    • 使用Windows的MySQL安装程序：

      1. 启动Windows的MySQL安装程序。MySQL安装程序仪表板打开。

      2. 单击重新配置 MySQL服务器的快速操作。使用Next和 Back来配置以下项目：

        • 在帐户和角色中，确认当前root帐户密码。

        • 在插件和扩展中，选中Enable X Protocol / MySQL as Document Store复选框。MySQL安装程序提供一个默认端口号并打开防火墙端口以进行网络访问。

        • 在应用服务器配置中，单击执行。

        • 单击完成关闭MySQL安装程序。

      3. 安装MySQL Shell。

    • 使用MySQL Shell：

      1. 安装MySQL Shell。

      2. 打开终端窗口（Windows上的命令提示符）并导航到MySQL二进制文件位置（例如， /usr/bin/在Linux上）。

      3. 运行以下命令： 

        mysqlsh -u user-h localhost --classic --dba enableXProtocol

    • 使用MySQL客户端程序：

      1. 打开终端窗口（Windows上的命令提示符）并导航到MySQL二进制文件位置（例如， /usr/bin/在Linux上）。

      2. 调用mysql命令行客户端： 

        mysql -u user-p

      3. 发表以下声明： 

        MySQL的> INSTALL PLUGIN mysqlx SONAME 'mysqlx.so';

        更换mysqlx.so用 mysqlx.dll的Windows。

        重要

        该mysql.session用户必须存在，你可以加载X插件。 mysql.session被添加到MySQL版本5.7.19中。如果您的数据字典是使用较早版本初始化的，则必须运行 mysql_upgrade过程。如果升级未运行，X插件无法启动，并显示错误消息尝试使用以下用户访问服务器时发生错误：mysql.session@localhost。确保用户存在于服务器中，并且在服务器更新后运行mysql_upgrade。。

      4. 安装MySQL Shell。

  3. 确认X插件已安装。

    当X插件安装正确时，当您使用以下命令之一查询服务器上的活动插件时，它会显示在列表中：

    • MySQL Shell命令： 

      mysqlsh -u user--sqlc -e“显示插件”

    • MySQL客户端程序命令： 

      mysql -u user-p -e“显示插件”

    如果您在安装X插件时遇到问题，或者想了解有关安装，配置或卸载服务器插件的其他方法，请参见 第5.5.1节“安装和卸载插件”。

mysqlxsys@localhost 用户帐号

安装X插件将创建一个 mysqlxsys@localhost用户帐户。如果由于某种原因创建用户帐户失败，则X插件安装也会失败。以下是关于 mysqlxsys@localhost用户帐户的用途以及创建失败时要执行的操作的说明。

X插件安装过程使用MySQL root用户为用户创建一个内部帐户 mysqlxsys@localhost。该 mysqlxsys@localhost帐户由X插件用于针对MySQL帐户系统对外部用户进行身份验证，并在特权用户请求时用于终止会话。该mysqlxsys@localhost帐户被创建为锁定，因此无法用于由外部用户登录。如果由于某种原因，MySQL root帐户不可用，则在启动X插件安装之前，必须mysqlxsys@localhost 通过在mysql命令行客户端中发出以下语句来手动创建用户 ： 

如果不存在，则创建用户mysqlxsys @ localhost IDENTIFIED WITH
mysql_native_password AS''ACCOUNT passwordLOCK;
授予选择在mysql.user到mysqlxsys @ localhost;
授予超级开* * *到mysqlxsys @ localhost;

卸载X插件

如果您想要卸载（停用）X插件，请在mysql命令行客户端中发出以下语句： 

UNINSTALL PLUGIN mysqlx;

不要使用MySQL Shell来发布前面的语句。它可以从MySQL Shell运行，但是会出现错误（代码1130）。另外，卸载插件会删除mysqlxsys用户。

      sudo apt-get安装mysql-shell

    • 更新MySQL APT存储库的包信息： 

      sudo apt-get update

      sudo apt-get install mysql-apt-config

      当在对话框中询问配置存储库时，请确保选择MySQL 5.7（这是默认选项）作为您想要的发行版系列，并启用MySQL预览包组件。

19.4.4.6创建和删除索引

索引用于快速查找具有特定字段值的文档。如果没有索引，MySQL必须从第一个文档开始，然后阅读整个集合以查找相关字段。收藏品越多，成本越高。如果集合很大并且特定字段上的查询很常见，则考虑在文档内的特定字段上创建索引。

例如，以下查询对索引的执行效果会更好： 

mysql-js> db.countryinfo.find("demographics.Population < 100")
... [ 输出删除 ]

该createIndex()方法创建一个索引，您可以将其定义为不唯一或唯一。使用该 field()方法链接应该编入索引的字段。该execute()方法是创建或删除索引所必需的。

在下面的示例中，方法的第一个参数 field()指定了人口统计信息对象中的Population字段，并且下一个参数指示该字段应该作为整数数值进行索引。最后一个参数指示该字段是否需要NOT NULL约束。如果值是 false，则该字段可以包含 NULL值。 

MySQL的-JS> db.countryinfo.createIndex("pop").

查询OK（0.04秒）

19.7.1使用X插件进行安全连接

本节介绍如何配置X Plugin以使用安全连接。有关更多背景信息，请参见 第6.4节“使用加密连接”。

X插件有自己的SSL设置，可以与MySQL服务器使用的不同。这意味着X插件可以配置不同于MySQL服务器的SSL密钥，证书和证书颁发机构文件。同样，X插件也有自己的SSL状态变量，这些变量独立于MySQL服务器SSL相关变量计算。默认情况下，X插件SSL配置取自 mysqlx_ssl_*变量，在描述 第19.7.2.2，“X插件系统变量和选项”。如果没有使用该提供的配置 mysqlx_ssl_*变量，X插件回退到使用MySQL服务器SSL系统变量。这意味着您可以选择为MySQL协议和X协议连接单独配置单独的SSL配置，或者通过配置ssl-*变量来共享MySQL协议和X协议连接之间的SSL配置 。

在安装了X插件的服务器上，要使用单独的SSL配置来配置MySQL协议和X协议连接，请在以下位置使用ssl-*和 mysqlx-ssl-*变量 my.cnf： 

的[mysqld]
SSL的CA = ca1.pem
SSL证书=服务器cert1.pem
SSL的密钥=服务器key1.pem

mysqlx-SSL-CA = ca2.pem
mysqlx-SSL证书=服务器cert2.pem
mysqlx的SSL密钥=服务器key2.pem

可用mysqlx_ssl_*变量镜像MySQL服务器中的SSL变量，因此在第6.4.1节“配置MySQL使用加密连接”中描述的用于配置MySQL服务器使用SSL的文件和技术 与配置X插件以使用安全连接相关。此规则的一个例外是mysqlx_ssl变量，该变量默认1配置为其他任何X协议SSL相关变量。换句话说，如果您配置了任何mysqlx_ssl_* 变量，则无需进行配置 mysqlx_ssl。

您可以使用tls_version 系统变量配置由X协议SSL连接使用的TLS版本。MySQL协议和X协议连接使用的TLS版本因此是相同的TLS版本。

每个连接的加密是可选的，但是特定的用户可能被迫使用X协议和MySQL协议连接加密。您可以通过发布GRANT带有REQUIRE选项的语句来 配置这样的用户 。有关更多详细信息，请参见 第13.7.1.4节“GRANT语法”。或者，所有X协议和MySQL协议连接都可以通过设置被强制使用加密require_secure_transport。

    • 如果地址是*，则如果服务器主机支持IPv6，则X Plugin在所有服务器主机IPv6和IPv4接口上接受TCP / IP连接，否则接受所有IPv4地址上的TCP / IP连接。使用此地址允许所有服务器接口上的IPv4和IPv6连接。该值是默认值。

19.7.3监控X插件

19.7.3.1 X插件的状态变量

本节介绍如何监视X插件。有两种可用的监视方法，使用性能模式表或状态变量。

19.7.3.1 X插件的状态变量

状态变量具有以下含义。

  • Mysqlx_address

    X Plugin绑定的网络地址。如果绑定失败，或者 skip_networking选项已被使用，则显示该值UNDEFINED。

  • Mysqlx_bytes_sent

    通过网络发送的字节数。

  • Mysqlx_bytes_received

    通过网络接收的字节数。

  • Mysqlx_stmt_execute_sql

    为SQL命名空间收到的StmtExecute请求的数量。

  • Mysqlx_stmt_execute_xplugin

    为X插件名称空间收到的StmtExecute请求的数量。

  • Mysqlx_stmt_create_collection

    收到的创建收集语句的数量。

  • Mysqlx_stmt_create_collection_index

    创建收集索引语句的数量。

  • Mysqlx_stmt_drop_collection

    收到的丢弃收集语句的数量。

  • Mysqlx_stmt_drop_collection_index

    收到的丢弃收集索引语句的数量。

  • Mysqlx_stmt_ensure_collection

    确保收集声明的数量。

  • Mysqlx_stmt_execute_mysqlx

    收到的命名空间设置为的StmtExecute消息的数量mysqlx。

  • Mysqlx_stmt_list_objects

    收到的列表对象语句的数量。

  • Mysqlx_stmt_enable_notices

    收到的启用通知语句的数量。

  • Mysqlx_stmt_disable_notices

    收到的禁用通知语句的数量。

  • Mysqlx_stmt_list_notices

    收到的清单通知报表数量。

  • Mysqlx_stmt_list_clients

    收到的列表客户端语句的数量。

  • Mysqlx_stmt_kill_client

    所收到的kill客户端语句的数量。

  • Mysqlx_stmt_ping

    收到的ping语句的数量。

  • Mysqlx_crud_insert

    接收的插入请求的数量。

  • Mysqlx_crud_update

    收到的更新请求的数量。

  • Mysqlx_crud_create_view

  • Mysqlx_crud_drop_view

  • Mysqlx_crud_find

  • Mysqlx_expect_open

  • Mysqlx_expect_close

  • Mysqlx_rows_sent

  • Mysqlx_notice_other_sent
