动态链接库管理 Ldconfig
admin 2016/05/23 2018-04-29

函数库

函数库用于将相似函数打包在一个单元中。然后这些单元就可为其他开发人员所共享，并因此有了模块化编程这种说法。
从模块中构建程序，Linux 支持两种类型的库，每一种库都有各自的优缺点。
1、静态库包含在编译时静态绑定到一个程序的函数。
2、动态库则不同，它是在加载应用程序时被加载的，而且它与应用程序是在运行时绑定的。使用共享库的方法有两种：您既可以在运行时动态链接库，也可以动态加载库并在程序控制之下使用它们。

管理命令

ldconfig命令用来管理动态运行库，让动态链接库为系统所共享。
ldconfig 命令在默认搜寻目录(/lib和/usr/lib)和动态库配置文件/etc/ld.so.conf内所列的目录下;搜索出可共享的动态 链接库(格式如前介绍,lib*.so*),进而创建出动态装入程序(ld.so)所需的连接和缓存文件。
缓存文件(/etc/ld.so.cache)此文件保存已排好序的动态链接库名字列表。
ldconfig通常在系统启动时运行；而当用户安装了一个新的动态链接库时,就需要手工运行这个命令，刷新系统的动态链接库索引。

命令用法

ldconfig [-v|--verbose] [-n] [-N] [-X] [-f CONF] [-C CACHE] [-r ROOT] [-l] [-p|--print-cache] [-c FORMAT] [--format=FORMAT] [-V] [- |--help|--usage] path...

-v或–verbose 列出动态链接库和连接的名字
-n 仅扫描指定的目录并不建立缓存
-N 不重建缓存文件(/etc/ld.so.cache);若未用-X选项,ldconfig照常更新文件的连接
-X 不更新文件的连接。若未用-N选项,则缓存文件正常更新
-f CONF 指定动态链接库的配置文件；系统默认为/etc/ld.so.conf
-C CACHE 生成的缓存文件为CACHE
-r ROOT 此选项改变应用程序的根目录为ROOT(是调用chroot函数实现的)
-l 专家模式,需要手工设置连接
-p或–print-cache 打印当前缓存文件所保存的所有共享库的名字
-c FORMAT 或 –format=FORMAT 此选项用于指定缓存文件所使用的格式,共有三种: ld(老格式),new(新格式)和compat(兼容格式,此为默认格式)
-V 此选项打印出ldconfig的版本信息,而后退出
– 或 –help 或 –usage 打印出其帮助信息,而后退出

注意

1、新增的库(/lib和/usr/lib目录下)，不需要修改稿/etc/ld.so.conf；需要调用ldconfig，重新生成缓存索引。
2、新增的库，如果不在默认目录，需要将目录写入/etc/ld.so.conf；然后再调用ldconfig，不然也会找不到。
3、第三种情况，修改系统变量LD_LIBRARY_PATH；也可以找到动态连接库。
4、ldconfig 重新生成系统动态链接库索引，与运行进程有关；和编译安装无关。
