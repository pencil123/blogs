Nginx Index 和 AutoIndex 页面索引
admin 2015/05/21 2018-04-20

Nginx 服务中ngx_http_index_module 模块用来处理以斜杠(‘/’)结尾的请求。这些请求也可以由ngx_http_autoindex_module和ngx_http_random_index_module模块处理。

http_index_module 模块

index 索引文件

Default: index index.html;
Context: http, server, location
模块:ngx_http_autoindex_module
设定用作为索引的文件名。文件名可以包含变量；检测索引按照设定的检测顺序。最后的一个文件列表元素可以设置为绝对路径。

http_autoindex_module 模块

当ngx_http_index_module模块找不到索引文件时，通常会将请求传递到ngx_http_autoindex_module模块。
ngx_http_autoindex_module模块绝对是否列出路径下的文件列表。

autoindex off|on

默认值：关闭
配置段：http,server,location

autoindex_exact_size on

默认为on，显示出文件的确切大小，单位是bytes。
改为off后，显示出文件的大概大小，单位是kB或者MB或者GB

autoindex_localtime off

默认为off，显示的文件时间为GMT时间。
改为on后，显示的文件时间为文件的服务器时间

例子 Example

虚拟主机
location / {
autoindex on;
autoindex_localtime on;
}

单独目录
location /down/ {
autoindex on;
}

虚拟目录
location /down/ {
alias /home/wwwroot/lnmp/test/;
autoindex on;
}
