Mysql连接线程相关
admin 2015/12/27 2018-03-26

thread_cache_size

mysql 连接线程缓存数量
mysql 每一个客户端连接，都需要使用一个连接线程。
当一个客户端连接断开后，mysql可以不关闭这个线程；缓存起来，用于新客户端连接的重用。
连接线程重用会提供mysql服务对新连接建立的响应速度。mysql服务会缓存的连接线程数量由thread_cache_size 配置。

mysql处理方式
当客户端断开连接时，如果线程数少于thread_cache_size，客户端的线程将被放入缓存中。
如果可能的话，重用使用高速缓存的线程来满足线程请求，并且只有当高速缓存为空时才创建新的线程。

设置环境
如果您有很多新的连接，可以增加此变量以提高性能。
通常情况下，如果你有一个好的线程实现，这不会提供显着的性能改进。
但是，如果您的服务器每秒看到数百个连接，则通常应将thread_cache_size设置为足够高，以便大多数新连接使用缓存线程。

设置参考
通过检查Connections和Threads_created状态变量之间的区别，可以看到线程缓存的有效性。
默认值基于以下公式，上限为100：8 +（max_connections / 100）

关于线程缓存状态参数：show global status
Threads_cached :当前线程缓存中有多少空闲线程。
Threads_connected :当前已建立的连接的数量，即连接线程数。
Threads_created :已创建线程的数量。
Threads_running :当前激活的（非睡眠状态）线程数。

线程资源使用设置参数

thread_stack

每个线程的堆栈大小。默认的192KB（对于64位系统是256KB）对于正常操作来说足够大。如果线程堆栈太小，则会限制服务器可以处理的SQL语句的复杂性，存储过程的递归深度以及其他耗费内存的操作。

thread_handling

服务器用于连接线程的线程处理模型。
允许的值是:
no-threads（服务器使用单个线程来处理一个连接）
one-thread-per-connection（服务器使用一个线程来处理每个客户端连接）

max-connections

default 151

同时进行的客户端连接的最大允许数量。 默认情况下，这是151。
mysqld实际上允许max_connections + 1个客户端连接。
额外的连接被保留供具有SUPER特权的帐户使用。通过向管理员授予SUPER权限(而不授予普通用户)，管理员可以连接到服务器，并使用SHOW PROCESSLIST来诊断问题。

最大连接数设置考量

MySQL支持的最大连接数取决于：
1、操作系统平台线程库的质量(代码的质量)
2、系统内存大小
3、每个连接数使用的内存
4、每个连接的工作负载
5、响应需要的时间
增加最大客户端连接数量会增加mysqld所需的文件描述符的数量。
相关设置：table_open_cache、–open-files-limit=count

Too many connections error

如果尝试连接到mysqld服务器时出现“连接错误太多”，则表示所有可用的连接正在被其他客户端使用。
