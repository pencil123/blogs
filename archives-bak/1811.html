Git 文件忽略:gitignore
admin 2017/01/28 2018-05-01

文件忽略机制是用于将“工作目录”中的部分文件不进行跟踪,不加入到“暂存区”。
git 版本控制有两种方法进行文件忽略；这里只说gitignore文件配置方法。

配置语法

1、以斜杠“/”开头，表示忽略“工作目录”下的某个目录
2、以星号“*”通配符，正则匹配多个字符
3、以问号“?”通配符，正则匹配单个字符
4、以方括号“[]”正则符号，包含单个字符的匹配列表
5、以叹号“!”正则符号，表示不忽略匹配到的文件或目录

补充：忽略文件中规则生效方式，是从上到下的。

实例

忽略“工作目录”下级test目录
/test/*
#只忽略“工作目录”下的test目录

忽略“工作目录”下所有test目录
test/*
#忽略“工作目录”下所有命名为test的目录
#如/hello/test/

只追踪少量文件
/*
!.gitignore
!/hello/
#忽略所有文件
#跟踪gitignore文件和hello跟目录

使用技巧

已跟踪的文件添加忽略
已跟踪的文件，即已经添加到“暂存区”；所以从“暂存区”中删除，再使用就不会报错
git rm --cached

命令：
git rm：删除Git的”暂存区”追踪文件记录，同时删除“工作目录”中文件
git rm –cached：删除Git的”暂存区”追踪文件记录，但是不删除“工作目录”中文件
