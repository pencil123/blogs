centos 配置无线网卡
admin 2016/07/20 2018-01-02

家庭网络在上班和晚上睡觉的时候，都是空闲不用的；这样太浪费交的网费了。于是就买了个迷你主机，利用空闲网络来下载电影或者其他的。
操作系统选择的是Centos6，使用samba服务将下载好的使用共享到windows。主要是linux系统比较稳定，并且字符界面占用物理资源小。
在配置过程中最复杂的就是无线网卡的配置，找了很多资料最终总是配置好了。但是因为无线速度和稳定问题，还是舍弃了；换成有线网络。

加载驱动

坑1、机器的无线网卡硬件比较旧，是BCM43225；官网已经都不提供网卡驱动的源码包了。在网上搜索安装到的文档都是要去官网下载驱动包进行编译安装，晕，怎么下啊？
坑2、在wiki中找到一篇文章 https://wiki.archlinux.org/index.php/broadcom_wireless 上面说驱动brcmsmac兼容支持bcm43225网卡。因为前面踩了很多坑，也不知道什么时候，已经将驱动brcmsmac安装成功了。
因为不熟悉无线网卡的使用，一直还以为没有解决驱动的问题那。
a、加载驱动
lsmod #查看已经加载的驱动
modprobe 命令
-a或–all 　载入全部的模块。
-c或–show-conf 　显示所有模块的设置信息。
-d或–debug 　使用排错模式。
-l或–list 　显示可用的模块。
-r或–remove 　模块闲置不用时，即自动卸载模块。
-t或–type 　指定模块类型。
-v或–verbose 　执行时显示详细的信息。
-V或–version 　显示版本信息。
-help 　显示帮助。
modprobe brcmsmac #加载brcmsmac驱动
b、开机加载驱动
在CentOS中自动加载内核模块，需要在/etc/sysconfig/modules/目录中增加一个脚本，在此脚本中加载所需的模块。
下面是我所用的一个名为brcmsmac.modules的脚本；注意：脚本权限为755
#! /bin/sh
/sbin/modinfo -F filename brcmsmac > /dev/null 2>&1
if [ $? -eq 0 ]; then
/sbin/modprobe brcmsmac
fi
上面的脚本首先检查brcmsmac模块文件是否存在，如果存在，则调用/sbin/modprobe命令加载此模块。
在网上有些文章建议在/etc/rc.local文件中增加modprobe命令，这样做在某些情况下工作会不正常。因为在启动时，/etc/rc.local在执行完rc.d目录中的程序后才执行，因此，如果在rc.d中的程序如果用到了所需加载的模块，就会出错。
另外有些文章提到修改/etc/modprobe.conf文件，其实是错误的。这个文件仅仅是定义一些内核模块的别名、以及一些特殊的加载、卸载命令。修改此文件并不能使模块在启动时被自动加载。

配置网卡

3、由上也就是驱动已经安装ok，下面就是对网卡的配置管理了。
a、安装无线网卡管理工具：yum install wireless wireless-tools -y
iwconfig #查看无线网卡信息；如果没有列出无线网卡信息，那么网卡驱动还是没有加载成功

打开无线网卡电源
iwconfig wlan0 txpower on
ifconfig wlan0 up #启动网卡
iwlist wlan0 scanning 查看无线信号
配置无线网络验证
安装wpa_supplicant 用于WPA加密
使用：
wpa_passphrase <SSID> [password] 。将输出内容添加到/etc/wpa_supplicant/wpa_supplicant.conf中
连接无线网络
wpa_supplicant -B -i wlan0 -D wext -c /etc/wpa_supplicant/wpa_supplicant.conf
-B 后台运行 -i 选择网卡 -D wext -c 配置文件地址 -d 调试
4、编辑网卡文件
新建网卡管理配置文件
touch /etc/sysconfig/network-scripts/ifcfg-wlan0
DEVICE=wlan0
TYPE=Wireless
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=dhcp
DEFROUTE=yes
PEERDNS=yes
PEERROUTES=yes
IPV4_FAILURE_FATAL=yes
IPV6INIT=no
WPA=”yes”
