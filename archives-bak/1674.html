Nginx服务URI 重写
admin 2015/05/23 2018-04-19

nginx的Rewrite规则采用PCRE(Perl Compatible Regular Expressions) Perl兼容的正则表达式的语法进行规则匹配。

使用场景

伪静态：动态url地址伪装成Html，便于搜索引擎抓取
唯一入口：一些编程框架MVC结构，默认请求统一交给一个文件进行处理。
301调整：一些由于域名、路径更改，需要做重定向跳转

Rewrite 语法

rewrite regex(匹配规则) replacement(重写规则) [flag](重写类型);
Default: —
Context: server, location, if
匹配规则：字符串或者正则表达式；匹配想要rewrite操作的URI
重写规则：对匹配URI重写的处理；可以使用$index表示匹配规则使用括号()捕获的分组
重写类型：定义重写操作是302,301或者在浏览器上不显示跳转。

  • last ：重写Rewrite后，会继续向下匹配；浏览器地址栏URL地址不变
  • break；规则匹配后，不再匹配后面的规则；浏览器地址栏URL地址不变
  • redirect：返回302临时重定向，浏览器地址会显示跳转后的URL地址
  • permanent：返回301永久重定向，浏览器地址栏会显示跳转后的URL地址

Rewrite Flag 说明

Nginx服务Rewrite重定向功能主要的falg有last break 和 redirect permanent

redirect 和permanent

临时重定向(redorect 302)和永久重定向(permanent 301)实际上就是将跳转操作通过返回头(Response Handers)发送到浏览器，浏览器获取新地址后重新发送请求。
所以这两个Rewrite操作通过将步骤分解，使整个过程清晰明了；也就没有什么复杂之处。

last 和 break

这两个Rewrite操作是在一次请求活动中处理有关重定向的问题。这里就涉及到修改NGINX接受到和初始化的请求变量(请求环境)和变量作用域问题。

使用说明

last :停止执行当前一系列Rewrite重写指令(此Location其他指令也不执行)；在Server指令中使用新的URI匹配新的Location。
stops processing the current set of ngx_http_rewrite_module directives and starts a search for a new location matching the changed URI。
break :停止执行当前一系列Rewrite重写指令;此Location下的其他指令可执行；此请求终结于此Locaton.
stops processing the current set of ngx_http_rewrite_module directives as with the break directive;

变量修改

rewrite /hello /world break|last;
Nginx服务为此次请求建立了一些列的变量；如$request_uri,$user_agent,$request_filename,$arg_name等。
Rewrite 指令会修改$uri;不会修改$request_uri。

使用注意问题

如果为静态资源设置Rewrite规则使用last和break；一般不会出现异常。
如果为动态资源(php)设置此规则；动态资源一般从$request_uri变量中获取请求文件；这样就会导致rewrite规则不生效。

解决方式


if ($request_uri = "/hello"){set $my_request_uri "/world";}
if ($request_uri != "hello"){set $my_request_uri $request_uri;}
location ~ \.php$ {
fastcgi_pass 127.0.0.1:9000;
fastcgi_index index.php;
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
include fastcgi_params;
fastcgi_param REQUEST_URI $my_request_uri; #对$request_uri变量重新赋值
}
补充说明：rewrite指令执行的顺序根据其在配置文件中出现的顺序确定，依次执行。

Return 语法

return code [text];
return code URL;
return URL;
Default: —
Context: server, location, if
停止处理，并返回指定的内容。
返回”状态码”和”响应主体”和重定向URL

简单例子

例子一
server {
rewrite /last.html /index.html last;
#/last.html 转到 /index.html；仍然进行后续匹配
-
rewrite /break.html /index.html break;
#/break.html 转到 /index.html ；不进行后续匹配
-
rewrite /redirect.html /index.html redirect;
#/redirect.html 302定向(临时)到 /index.html
-
rewrite /permanent.html /index.html permanent;
#/permanent.html 301定向(永久)到 /index.html
-
rewrite ^/html/(.+?).html$ /post/$1.html permanent;
#/html/*.html 301定向到 /post/*.html
-
rewrite ^/search\/([^\/]+?)(\/|$) /search.html?keyword=$1 permanent;
#/search/key 301定向到 /search.html?keyword=key
-
rewrite ^/archives\/(\d+)$ /archives/$1.html permanent;
#/archives/数字 301定向到 /archives/数字.html
}

例子二
server {
location / {
rewrite /last/ /q.html last;
}
if (!-f $request_filename){
rewrite . /index.php last;
}}
#Rewrite 操作被location 和if 操作约束

例子三
location ^~ /login/ {
return 301 https://$host$requst_uri;
#login 下的URL进行加密传输
}

rewrite_log on | off;

记录日志
Default:
rewrite_log off;
Context: http, server, location, if
启用或禁用将ngx_http_rewrite_module 模块指令处理结果记录到该级别的error_lognotice。
