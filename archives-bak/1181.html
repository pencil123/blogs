Beautiful Soup XML数据解析
admin 2017/08/04 2018-05-25

Beautiful Soup提供一些简单的、python式的函数用来处理导航、搜索、修改分析树等功能。它是一个工具箱，通过解析文档为用户提供需要抓取的数据，因为简单，所以不需要多少代码就可以写出一个完整的应用程序。
Beautiful Soup自动将输入文档转换为Unicode编码，输出文档转换为utf-8编码。你不需要考虑编码方式，除非文档没有指定一个编码方式，这时，Beautiful Soup就不能自动识别编码方式了。然后，你仅仅需要说明一下原始编码方式就可以了。

解析器

Beautiful Soup支持Python标准库中的HTML解析器,还支持一些第三方的解析器,其中一个是 lxml。另一个可供选择的解析器是纯Python实现的 html5lib , html5lib的解析方式与浏览器相同。
$ pip install lxml
$ pip install html5lib

四大对象种类

Beautiful Soup将复杂HTML文档转换成一个复杂的树形结构,每个节点都是Python对象,所有对象可以归纳为4种:

Tag 标签(HTML语言中的标签元素)存在很多方法和属性
NavigableString 遍历的字符串（字符串常被包含在tag内.BS用NavigableString类来包装tag中的字符串）
BeautifulSoup 一个文档的全部内容(大部分时候,可以把它当作 Tag 对象)
Comment 注释及特殊字符串（文档的注释部分）

Tag 对象属性

.content 以列表的形式列出直接子节点
.children 子节点列表生成器，可以用于遍历直接子节点
.descendants 上面两个属性只列出直接子节点；这个可以列出所有子节点
.string 节点内容（当tag对象中只有一个子节点）
.strings 多个节点内容遍历获取 （get_text()）
.stripped_strings 多节点内容 除去多余空白内容
.parent 获取某个元素的父节点
.parents 以递归得到元素的所有父辈节点
.next_sibling 下一个节点（兄弟节点）
.previous_sibling 上一个节点（兄弟节点）
.next_siblings 后面的节点迭代输出（兄弟节点）
.previous_siblings 前面的节点迭代输出（兄弟节点）
.next_element 当前节点的下一个解析对象
.previous_element 当前节点的上一个解析对象
.next_elements 迭代向前访问文档的解析对象
.previous_elements 迭代向后访问文档的解析对象

Tag 对象方法

find_all( name , attrs , recursive , text , **kwargs )
搜索当前tag的所有tag子节点,并判断是否符合过滤器的条件;返回一个列表

find( name , attrs , recursive , text , **kwargs )
搜索当前tag对象，只返回一个匹配元素

find_parents() find_parent()
搜索当前节点的父辈节点

find_next_siblings() find_next_sibling()
对当 tag 的所有后面解析的兄弟 tag 节点进行迭代,返回所有|第一个符合条件的后面的兄弟节点

find_previous_siblings() find_previous_sibling()
对当前 tag 的前面解析的兄弟 tag 节点进行迭代, 返回所有|第一个符合条件的前面的兄弟节点

find_all_next() find_next()
对当前 tag 的之后的 tag 和字符串进行迭代, 返回所有|第一个符合条件的节点

find_all_previous() 和 find_previous()
对当前节点前面的 tag 和字符串进行迭代,返回所有|第一个符合条件的节点

补充

1、Beautiful Soup 支持大部分的CSS选择器。
2、Beautiful Soup的强项是文档树的搜索,但同时也可以方便的修改文档树。
3、对修改的文档树进行输出方式

编码问题

任何HTML或XML文档都有自己的编码方式,比如ASCII 或 UTF-8,但是使用Beautiful Soup解析后,文档都被转换成了Unicode。

Beautiful Soup用了 编码自动检测 子库来识别当前文档编码并转换成Unicode编码.
BeautifulSoup 对象的 .original_encoding 属性记录了自动识别编码的结果。

编码自动检测 功能大部分时候都能猜对编码格式,但有时候也会出错。有时候即使猜测正确,也是在逐个字节的遍历整个文档后才猜对的,这样很慢。
如果预先知道文档编码,可以设置编码参数来减少自动检查编码出错的概率并且提高文档解析速度.在创建 BeautifulSoup 对象的时候设置 from_encoding 参数。
补充
如果中文页面编码是gb2312，gbk，在BeautifulSoup构造器中传入fromEncoding=”gb18030″参数即可解决乱码问题，即使分析的页面是utf8的页面使用gb18030也不会出现乱码问题！
不过当页面编码为“iso-8859-1”时，还是会出现乱码。根本的解决方法还是根据具体页面编码，使用from_encoding参数传输给beautifulsoup。
