MySQL 日期和时间函数
admin 2015/12/21 2018-04-03

MySQL 5.7版本中，官网列出了60个与日期和时间相关的函数。

函数主要包括：

当前时间(不同的格式)
时间计算(过去或将来的时间)

在查询执行开始时，每个查询只返回一次返回当前日期或时间的函数。这意味着对一个函数（如NOW()单个查询）的多次引用 总是会产生相同的结果。（对于我们而言，单一的查询还包括对存储程序的调用（存储程序，触发器或事件）以及所有子程序由该程序调用。）这个原则也适用于CURDATE()， CURTIME()， UTC_DATE()， UTC_TIME()， UTC_TIMESTAMP()，和他们的任何同义词。

The CURRENT_TIMESTAMP(), CURRENT_TIME(), CURRENT_DATE(), and FROM_UNIXTIME() functions return values in the connection’s current time zone, which is available as the value of the time_zone system variable.

MySQL 中有5个函数获取当前系统时间

NOW():返回时间，格式如：2012-09-23 06:48:28
CURDATE():返回时间的日期，格式如：2012-09-23
CURTIME():返回时间，格式如：06:48:28
UNIX_TIMESTAMP():返回时间整数戳，如：1348408108
SYSDATE():返回时间，格式和time()函数相同

前四个函数返回的时间都是基于整个SQL 语句执行的时间。
SYSDATE()函数访问的时间是基于SYSDATE()函数执行的时间。

示例

mysql> select now(),sleep(2),now();
+---------------------+----------+---------------------+
| now() | sleep(2) | now() |
+---------------------+----------+---------------------+
| 2018-04-03 14:50:26 | 0 | 2018-04-03 14:50:26 |
返回的时间相同，为整个SQL 语句开始执行的时间。
mysql> select sysdate(),sleep(2),sysdate();
+---------------------+----------+---------------------+
| sysdate() | sleep(2) | sysdate() |
+---------------------+----------+---------------------+
| 2018-04-03 14:50:41 | 0 | 2018-04-03 14:50:43 |
返回的时间不同,为SYSDATE()函数执行的时间。

可以在配置文档中，配置sysdate_is_now 将SYSDATE函数设置为NOW函数的别名。避免SYSDATE函数时间问题。
