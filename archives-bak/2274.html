Data Handling

arg_separator.output string
The separator used in PHP generated URLs to separate arguments.
在PHP中使用的分隔符生成URL来分隔参数。

arg_separator.input string
List of separator(s) used by PHP to parse input URLs into variables.
PHP使用的分隔符列表将输入URL解析成变量。

variables_order string
Sets the order of the EGPCS (Environment, Get, Post, Cookie, and Server) variable parsing. For example, if variables_order is set to “SP” then PHP will create the superglobals $_SERVER and $_POST, but not create $_ENV, $_GET, and $_COOKIE.
设置EGPCS（环境，获取，发布，Cookie和服务器）变量解析的顺序。 例如，如果将variables_order设置为“SP”，那么PHP将创建超级页面$ _SERVER和$ _POST，但不会创建$ _ENV，$ _GET和$ _COOKIE。
Setting to “” means no superglobals will be set.
设置为“”表示不设置超级页面。

request_order string
This directive describes the order in which PHP registers GET, POST and Cookie variables into the _REQUEST array. Registration is done from left to right, newer values override older values.
该指令描述了将GET，POST和Cookie变量注册到_REQUEST数组中的顺序。 注册从左到右完成，较新的值覆盖旧值。

register_argc_argv boolean
Tells PHP whether to declare the argv & argc variables (that would contain the GET information).
告诉PHP是否声明argv＆argc变量（将包含GET信息）。

enable_post_data_reading boolean
Disabling this option causes $_POST and $_FILES not to be populated.
禁用此选项将导致$ _POST和$ _FILES不被填充。
The only way to read postdata will then be through the php://input stream wrapper. This can be useful to proxy requests or to process the POST data in a memory efficient fashion.
读取postdata的唯一方法就是通过php：//输入流包装器。 这对于代理请求或以高效的存储器来处理POST数据是有用的。

auto_globals_jit boolean
When enabled, the SERVER and ENV variables are created when they’re first used (Just In Time) instead of when the script starts.
启用时，SERVER和ENV变量将在首次使用（即时）时创建，而不是脚本启动时创建。
If these variables are not used within a script, having this directive on will result in a performance gain.
如果脚本中没有使用这些变量，那么执行此指令将导致性能提升。

post_max_size integer
Sets max size of post data allowed.

register_globals boolean
Whether or not to register the EGPCS (Environment, GET, POST, Cookie, Server) variables as global variables.

auto_prepend_file string
Specifies the name of a file that is automatically parsed before the main file. The file is included as if it was called with the require function, so include_path is used.
指定在主文件之前自动解析的文件的名称。 包含该文件，就像使用require函数调用一样，因此使用include_path。

auto_append_file string
Specifies the name of a file that is automatically parsed after the main file. The file is included as if it was called with the require function, so include_path is used.
指定在主文件之后自动解析的文件的名称。 包含该文件，就像使用require函数调用一样，因此使用include_path。

default_mimetype string
PHP’s built-in default media type is set to text/html.

default_charset string
PHP always outputs a character encoding by default in the Content-type: header. To disable sending of the charset, simply set it to be empty.
