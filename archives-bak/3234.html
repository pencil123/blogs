kill 发送管理信号
admin 2016/03/19 2018-01-21

Linux 系统用于对进程发送管理信号的部分命令：

kill terminate a process 终止一个进程
killall kill processes by name 根据名字杀掉所有进程
pkill [pgrep] look up or signal processes based on name and other attributes 根据名字和其它属性查看或者发出进程信号
skill send a signal or report process status 发送一个信号或者报告进程状态

kill 对进程发送特定信号

不同服务程序对这些控制信号有不同的处理机制。
usage: kill [-s signal|-p] [–] pid…
-l <信号>：若不加<信号>选项，则-l参数会列出全部的信号
-u：指定用户

killall 对所有进程发送特定的信号

使用参数可以对进程做其他处理
-I –ignore-case: 忽略大小写
-i –interactive: 交互式终止进程
-q –quiet: 关闭命令执行回显

pkill 识别进程属性发送信号

pkill [-signal] [-fvx] [-n|-o] [-P ppid,...] [-g pgrp,...]
[-s sid,...] [-u euid,...] [-U uid,...]
[-G gid,...] [-t term,...] [pattern]

-P：指定父进程号发送信号；
-g：指定进程组；
-t：指定开启进程的终端。

踢出系统登陆用户
pkill -9 -t pts/0

关联命令:pgrep

skill 发送信号和报告进程状态

skill [signal to send] [options] process selection criteria

常规参数
-f：快速模式；
-i：交互模式，每一步操作都需要确认；
-v：冗余模式；
-w：激活模式；

进程选择参数
-t：进程|用户终端
-u：用户名
-p：进程号
-c：进程名
