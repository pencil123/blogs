Redis动态修改配置
admin 2016/09/15 2018-03-28

redis可以在不停止和重启服务的情况下，修改服务配置。这就使用到redis终端中的config命令。

修改过程：

查看当前配置：config get config_setting_name
修改配置参数：config set config_setting_Name new_config_value
将修改写入config文件中：config rewrite

1、获取配置参数

》config get loglevel
1) "loglevel"
2) "notice"

》config get log*
1) "logfile"
2) "/var/log/redis.log"
3) "loglevel"
4) "notice"

所有的配置参数
》config get *

2、修改配置参数

支持大多数参数，部分不支持
》config get dir
1) "dir"
2) "/usr/local/bin"
》config set dir /usr/
OK
》config get dir
1) "dir"
2) "/usr"

3、重写config文件

上面set修改的参数写入config文件中
》config rewrite

重写会以非常保守的方式进行

1、原有 redis.conf 文件的整体结构和注释会被尽可能地保留。
2、如果一个选项已经存在， 那么重写会在选项原本所在的位置（行号）上进行。
3、如果一个选项不存在， 并且该选项被设置为默认值，那么重写程序不会处理。
4、如果一个选项不存在， 并且该选项被设置为非默认值，那么这个选项将被添加到redis.conf文件的末尾。
5、未使用的行会被留白。
比如说， 如果你在原有 redis.conf 文件上设置了数个关于 save 选项的参数， 但现在你将这些 save 参数的一个或全部都关闭了， 那么这些不再使用的参数原本所在的行就会变成空白的。
6、即使启动服务器时所指定的 redis.conf 文件已经不再存在， CONFIG REWRITE 命令也可以重新构建并生成出一个新的 redis.conf 文件
