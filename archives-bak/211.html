主要分为：es配置和日志的配置

es配置：
——Cluster
cluster.name: elk
—–Node
node.name: elk0-info
node.attr.rack: r1
——-Paths
path.data: /data/elastic/data
path.logs: /data/elastic/logs
———-Memory
bootstrap.memory_lock: false
#是否将JVM的内存锁定
bootstrap.system_call_filter: false
———–Network
network.host: IP addr
http.port: 9200
transport.tcp.compress: false
————Discovery
discovery.zen.ping.unicast.hosts: [“elk1-info”, “elk2-info”,”elk0-info”]
#检查集群的节点
discovery.zen.minimum_master_nodes: 2
#防止脑裂（num/2）+1
———Gateway
gateway.recover_after_nodes: 3
———-Various
action.destructive_requires_name: true
———–Index
index.number_of_shards: 3
index.number_of_replicas: 2
refresh_interval： 5s

bootstrap检查
这些引导检查检查各种Elasticsearch和系统设置，并将它们与Elasticsearch操作安全的值进行比较

开发模式和生产模式
es如果在单机上运行（哪怕启动了多个实例），则被认为为开发模式；否则为生产模式。生成模式中bootstrap检查很更严格。

Heap size check 堆大小检查
File descriptor check 文件描述符检查
Memory lock check 内存锁定检查
Maximum number of threads check 最大线程数检查
Maximum size virtual memory check 最大的虚拟内存检查
Max file size check 最大文件大小检查
Maximum map count check 最大内存映射数量检查
Client JVM check 客户端JVM检查
Use serial collector check 使用串行收集器检查
System call filter check 系统调用过滤器检查
OnError and OnOutOfMemoryError checks OnError和OnOutOfMemoryError检查
Early-access check 提前检查
G1GC check G1GC检查

操作系统设置：
设置JVM堆大小
禁用交换
增加文件描述符
确保足够的虚拟内存
确保足够的线程
