Nginx If 判断语句使用
admin 2015/07/23 2018-04-20

if (condition) { … } 判断

Default: —
Context: server, location
用于检查一个条件是否符合；
如果符合，则执行大括号中的语句；
不支持嵌套，不支持多条件&&和||处理。

判断运算符

一个变量名；如果变量的值为空字符串或“ 0”; 则为false ;

  • “ =”和“ !=”运算符比较一个变量与一个字符串;
  • “ ~”（区分大小写匹配）和“ ~*”（用于不区分大小写的匹配）运算符，将变量与正则表达式进行匹配。正则表达式可以包含可用于稍后在$1.. $9变量中重复使用的捕获。负操作员“ !~”和“ !~*”也可用。如果正则表达式包含“ }”或“ ;”字符，那么整个表达式应该用单引号或双引号括起来。
  • “ -f”和“ !-f”操作符检查文件存在;
  • “ -d”和“ !-d”操作符检查目录存在;
  • “ -e”和“ !-e”操作符检查文件，目录或符号链接存在;
  • “ -x”和“ !-x”运算符检查可执行文件。

示例


# 如果文件不存在则返回400
if (!-f $request_filename) {
return 400;
}

# 如果host不是example.com，则301到example.com中
if ( $host != 'example.com' ){
rewrite ^/(.*)$ https://example.com/$1 permanent;
}

# 如果请求类型不是POST则返回405
if ($request_method = POST) {
return 405;
}

# 如果参数中有 a=1 则301到指定域名
if ($args ~ a=1) {
rewrite ^ http://example.com/ permanent;
}

在某种场景下可结合location规则来使用，如：
# 访问 /test.html 时
location = /test.html {
# 默认值为xiaowu
set $name xiaowu;
# 如果参数中有 name=xx 则使用该值
if ($args ~* name=(\w+?)(&|$)) {
set $name $1;
}
# 301
rewrite ^ /$name.html permanent;
}

/test.html => /xiaowu.html
/test.html?name=ok => /ok.html?name=ok
