强制关机|重启
admin 2016/10/21 2018-06-06

I have a linux machine dedicated to MySQL server with a pretty high load. Today I woke up and was terrified to see that database server is down. I could connect to it via SSH, but it was responding with bus error to each and every command。
我有一个用于运行mysql数据库的linux机器。今天发现系统资源（CPU,RAM）不足,导致数据库停到了。连接上ssh,执行每个命令都提示总线错误;

So I think I need reboot it,but reboot or ·init 0· these commands can not work.
使用reboot,init 0命令进行重启|关机； 也不起作用。 解决方法：
I’m not quite sure if this is what you’re looking for (since it’s like pressing the reset button), but this has helped me in situations where a server refused to reboot:
当系统拒绝重启时，可以使用一下操作：

echo 1 > /proc/sys/kernel/sysrq
echo b > /proc/sysrq-trigger

The first line is for enabling SysRq, and the second one is for rebooting.
第一条命令 用于激活sysrq组合键功能；
第二条命令 用于重启。

SysRq组合键说明

使用sysrq组合键是了解系统目前运行情况的最佳方式。
如果系统出现挂起的情况或在诊断一些和内核相关，比较怪异，比较难重现的问题的时候，使用sysrq键是个比较好的方式。
sysrq功能被打开后，有几种sysrq事件能被触发。不同的内核版本可能会有些不同。
但有一些是共用的:

m 导出关于内存分配的信息
t 导出线程状态信息
p 导出当前CPU寄存器信息和标志位的信息
c 故意让系统崩溃(在使用netdump或diskdump的时候有用)
s 即时同步所有挂载的文件系统
u 即时重新挂载所有的文件系统为只读
b 即时重新启动系统
o 即时关机(如果机器设置并支持此项功能)
