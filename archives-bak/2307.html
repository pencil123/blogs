代码回滚 Git
admin 2017/01/28 2018-05-01

git中用于代码回滚的有reset、checkout和revert命令。

1、reset将当前分支的末端指向另一个提交。这样可以移除当前分支的一些提交。
2、checkout将HEAD移到一个新的分支，然后更新工作目录。
3、checkout可以对还未提交的修改文件，直接恢复到未修改前（上一个commit）。
4、revert将之前的提交进行反写，然后创建新的提交
5、reset操作的是提交；checkout操作的主要是分支
6、reset和checkout两个命令可以针对单个文件进行回滚。

未提交的修改

在工作区修改文件，并没有进行提交操作；使用checkout命令将文件恢复到未修改前的状态。
单个文件
git checkout [head] readme.txt
所有txt文件
git checkout [head] *.txt
所有文件
git checkout [head] *

已提交的修改

重写提交历史reset

将当前分支的末端指向之前的某一个提交；这个特定提交之后的提交就变为悬挂提交。
悬挂提交会在下次 Git 执行垃圾回收的时候被删除。

通过传入这些标记来修改你的暂存区或工作目录：

–soft 暂存区和工作目录都不会被改变
–mixed 默认选项。暂存区和你指定的提交同步，但工作目录不受影响
–hard 暂存区和工作目录都同步到你指定的提交

#退回到上一个提交
git reset HEAD^

#将单个文件回退到上个提交
git reset HEAD^ a.py

#向前回退到第3个提交
git reset –soft HEAD~3

#回退到某个提交
git reset 057d

保留历史提交的回滚

Revert 撤销一个提交的同时会创建一个新的提交。这是一个安全的方法，因为它不会重写提交历史。
相比 git reset，它不会改变现在的提交历史。因此，git revert 可以用在公共分支上，git reset 应该用在私有分支上。
你也可以把 git revert 当作撤销已经提交的更改，而 git reset HEAD 用来撤销没有提交的更改。
就像 git checkout 一样，git revert 也有可能会重写文件。所以，Git 会在你执行 revert 之前要求你提交或者缓存你工作目录中的更改。

反转提交
git revert HEAD
#回退到上一次提交的状态，按照某一次的commit完全反向的进行一次commit.

已创建新的分支

使用checkout回滚到之前的分支。

查看提交日志

git log
#显示从最近到最远的提交日志

git log --pretty=oneline
#缩减版的提交日志记录

git reflog
#查看所有对版本的所有操作记录(包括commit和reset的操作),包括已经被删除的commit记录
