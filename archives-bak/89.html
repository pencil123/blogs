系统最大文件数
admin 2015/08/18 2018-01-11

操作系统使用文件描述符处理文件系统文件以及伪文件—如连接和监听套接字。

优点和缺点

最大连接数影响到服务运行情况：
1、单个进程能同时访问的最大文件个数
2、单个nginx_worker进程能够接受的连接
3、网络内核管理系统端口的范围

最大文件数设置过大的缺点：
系统内核或其他运行服务，如果继承最大文件数，则会导致服务运行要申请更大的内存或服务配置优化降级。

更改配置

1、修改配置，永久生效

文件：/etc/security/limits.d/90-nproc.conf
* soft nproc 10240
root soft nproc unlimited

文件：/etc/security/limits.conf
* soft nofile 8000
* hard nofile 10240
* soft nproc 8000
* hard nproc 10240

* 代表针对所有用户

soft为软设定：如果超出这个设定值会发出报警；enforcing the soft limits
hard为硬设定：不能超出这个设定值；enforcing hard limits
nproc 是代表最大进程数；nproc – max number of processes
nofile 是代表最大文件打开数；max number of open file descriptors

文件修改后，重新登陆后就会生效。

修改内核文件
echo “fs.file-max=6553500” >>/etc/sysctl.conf

2、修改系统内核，多终端生效
修改/proc内存参数
echo 100000 > /proc/sys/fs/file-max

修改内核参数
sysctl -w fs.file-max=6553500

3、命令修改，当前终端有效
ulimit 命令
ulimit -Hn 102401
ulimit -Sn 102400

-H ：hard limit ，严格的设定，必定不能超过这个设定的数值；
-S ：soft limit ，警告的设定，可以超过这个设定值，但是若超过则有警告讯息。

查看当前最大文件数

查询内存文件
cat /proc/sys/fs/file-max

使用命令
ulimit -Sn
ulimit -Hn

查看内核参数
sysctl fs.file-max
