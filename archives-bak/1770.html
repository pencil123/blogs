Git 标签操作
admin 2017/09/26 2018-05-01

Git提供了对版本打标签（tag）的功能。
标签功能实际上是指向某个Commit的指针，是某个Commit后代码的快照。和分支指针不同的是标签指针不随着后续的Commit而移动。

标签功能的作用
当发布版本时，如果新版本的代码指向某个Commit；使用标签功能方便管理。毕竟Commit自动生产的标识，是无序的字符串。

创建标签

git tag
#用于新建一个标签，默认为HEAD，也可以指定一个commit id
git tag -a -m “blablabla…”
可以指定标签信息
git tag -s -m “blablabla…”
可以用PGP签名标签；

查看标签

git tag
可以查看所有标签。
git show
可以看到PGP签名信息：

检出标签

git checkout tag_name

推送到远程

git push origin
如果要推送某个标签到远程，使用命令
git push origin –tags
一次性推送全部尚未推送到远程的本地标签：

删除标签

git tag -d v0.9
Deleted tag ‘v0.9’ (was 6224937)
删除本地

git push origin :refs/tags/v0.9
To git@github.com:michaelliao/learngit.git
– [deleted] v0.9
删除远程标签
先从本地删除；然后push到远程
