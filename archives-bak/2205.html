Comet 彗星
admin 2017/05/24 2017-10-25

Comet is a web application model in which a long-held HTTP request allows a web server to push data to a browser, without the browser explicitly requesting it.
Comet是web应用程序模块；它将建立一个长期持有的HTTP请求，允许web 服务器主动推送数据到浏览器；而不需要浏览器主动请求。

Comet is an umbrella term, encompassing multiple techniques for achieving this interaction.
Comet是一个总括术语，涵盖了实现这种交互的多种技术。

All these methods rely on features included by default in browsers, such as JavaScript, rather than on non-default plugins.
这些所有的方式都依赖于浏览器中默认的功能，如JavaScript,而不是默认插件。

The Comet approach differs from the original model of the web, in which a browser requests a complete web page at a time.
Comet方法和WEB的原始模型不同，Comet方法中浏览器一次请求，然后由服务器推送数据。web原始模型由浏览器发起每个资源的请求。

The use of Comet techniques in web development predates the use of the word Comet as a neologism for the collective techniques.
Comet作为这种技术的集体技术的新词使用。

Comet is known by several other names, including Ajax Push,Reverse Ajax,Two-way-web,HTTP Streaming, and HTTP server push among others.

The term Comet is not an acronym, but was coined by Alex Russell in his 2006 blog post Comet: Low Latency Data for the Browser.
Comet这个术语的由来，不是首字母的缩写，而是从Alex博士的文章《Comet：浏览器的低延迟数据》来的。

实现：
数据流：隐藏iframe;XMLHttpRequest
长轮询Ajax：XMLHttpRequest长轮询，脚本标签长轮询。

替换：
在HTML5标准中，定义了客户端和服务器通讯的WebSocket方式，在得到浏览器支持以后，WebSocket将会取代Comet成为服务器推送的方法，目前chrome、Firefox、Opera、Safari等主流版本均支持，Internet Explorer从10开始支持。
