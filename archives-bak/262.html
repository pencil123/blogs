es 数据修改
admin 2016/06/11 2016-06-11

不少人使用ELK对数据进行收集分析；但当对数据进行修改的时候，就有些困难了。当然网上也有很多相关的帖子和解决方法。
这里我列一下我碰到的需求，和解决方法：
需求：因为刚开始建立ELK时，没有规划好，所有平台的nginx日志都存储到同一个索引中了（[logstash-]YYYY.MM.DD）；这样很不方便管理。所以要将原来的数据根据平台不同划分开来。
解决方法：使用logstash处理；将INPUT和OUTPUT都配置为es，在读出和写入的中间过程做好筛选就可以了。
配置logstash：
input {
elasticsearch {
hosts => [“127.0.0.1:9200”]
index => “logstash-2016.06.07”
query => ‘{“query”:{“match”:{“type”:”keyword”}}}’
}
}
output {
elasticsearch {
hosts => [“127.0.0.1:9200”]
index => “newindex-2016.06.07”
}
}

运行就可以简单实现数据修改处理了。
