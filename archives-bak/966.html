Nginx 获取真实的访问IP
admin 2015/05/04 2018-04-22

本文用来说明Nginx 服务怎么才能够记录到访问者的真实IP地址。这个问题看似很容易，但是在使用应用中有很多细节需要注意。
Nginx 服务记录访问日志，主要是为了排查问题和做日志分析。做这两件事情都需要考虑到WEB服务器的架构。在WEB服务架构中，一般会涉及到多台WEB服务器的高可用和代理服务器。这样一次访问过程，可能会记录不止一次日志，也不确定会记录到哪一台WEB服务器上。这样如何获取一份完整的访问日志呐？我选择以核心服务器记录的日志为准。
这样的话，怎么设置才能是核心服务器记录到访问者真实的IP地址，就是要解决的问题了。

前提补充

X-Forwarded-For:简称XFF头，它代表客户端，也就是HTTP的请求端真实的IP.
只有在通过了HTTP 代理或者负载均衡服务器时才会添加该项。它不是RFC中定义的标准请求头信息，在squid缓存代理服务器开发文档中可以找到该项的详细介绍。
标准格式如下：X-Forwarded-For: client1, proxy1, proxy2。

代理服务器和CDN设置

代理服务器和CDN服务器需要将用户的IP地址和经过的代理服务器的IP地址传输到后端。
设置用来存储这些IP地址的请求Header字段；一般设置为X-Forwarded-For。

相关指令:proxy_set_header

重新定义或添加Header行到转发给后端服务器的请求信息中。
语法 ：proxy_set_header header value
他的值可以是文本、变量或者是文本和变量的组合。
当代理没有对指定和header行进行修改时，那么这个header行将从上级请求中继承。

注意事项：

proxy_set_header 设置header行，默认不支持下划线_;
如果事情需要配置underscores_in_headers on;
默认值：off
使用字段：http, server
是否允许在header的字段中带下划线

嵌入式变量:$proxy_add_x_forwarded_for

$remote_addr变量追加在客户端请求头“X-Forwarded-For”字段后，以逗号分割。如果客户端请求头中不存在“X-Forwarded-For”字段，则$ proxy_add_x_forwarded_for变量等于$ remote_addr变量。
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

示例

proxy_set_header -> Host -> $host; #请求的域名
proxy_set_header -> X-Real-IP -> $remote_addr; #连接此服务器的IP地址（用户真实IP）
proxy_set_header -> X-Forwarded-For -> $proxy_add_x_forwarded_for; #客户端ip和代理ip

核心服务器配置

使用ngx_http_realip_module模块，Nginx编译时添加–with-http_realip_module参数。
将之前的所有代理服务器和CDN服务器IP地址过滤掉。设定从固定的请求header字段过滤，并设定那些代理服务器IP过滤掉，这样就可以得到真实的用户IP地址。

set_real_ip_from address | CIDR | unix:

核心服务器前端的代理服务器ip或者CDN服务器IP段
Default: —
Context: http, server, location
Defines trusted addresses that are known to send correct replacement addresses. If the special value unix: is specified, all UNIX-domain sockets will be trusted.

real_ip_header field | X-Real-IP | X-Forwarded-For | proxy_protocol;

设置检索IP地址的header头字段
Default:real_ip_header X-Real-IP;
Context: http, server, location
Defines the request header field whose value will be used to replace the client address.
The X-Real-IP and X-Forwarded-For parameters may contain an optional port (1.11.0). The address and port should be specified according to RFC 3986.
The proxy_protocol parameter (1.5.12) changes the client address to the one from the PROXY protocol header. The PROXY protocol must be previously enabled by setting the proxy_protocol parameter in the listen directive.

real_ip_recursive on | off;

递归排除IP
Default:real_ip_recursive off;
Context: http, server, location
This directive appeared in versions 1.3.0 and 1.2.1.
If recursive（递归的） search is disabled, the original client address that matches one of the trusted addresses is replaced by the last address sent in the request header field defined by the real_ip_header directive. If recursive search is enabled, the original client address that matches one of the trusted addresses is replaced by the last non-trusted address sent in the request header field.

示例

location = /
{
set_real_ip_from 192.168.50.0/24;
set_real_ip_from 61.22.22.22;
set_real_ip_from 121.207.33.33;
set_real_ip_from 127.0.0.1;
real_ip_header X-Forwarded-For;
real_ip_recursive on;
fastcgi_pass unix:/var/run/phpfpm.sock;
fastcgi_index index.php;
include fastcgi.conf;
}
