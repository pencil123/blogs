Squid 进程管理命令：Squid和Squidclient

部分参数

-a port 指定http_port的端口号
-d level 将调试信息写到标准错误
-f configfile 指定配置文件
-k function reconfigure 重新读取配置文件
rotate 滚动日志
shutdown 关闭服务
interrupt 立即关闭
kill 关闭的最后保证
debug 完全调试模式
check 检查运行的进程
parse 检查配置文件是否有错误
-s 激活将日志记录到syslog进程
-u port 指定ICP端口
-z 初始化交换目录（缓存目录）
-D 禁止初始化DNS测试
-F 在重建存储元数据前，拒绝所有请求
-N 阻止Squid进程后台运行

SquidClient 命令使用

squidclient是一个提供用于检索URL的命令行界面的工具。
运行状态查看可用参数：squidclient -p 80 mgr:
状态信息：squidclient -p 80 mgr:info
内存信息：squidclient -p 80 mgr:mem
磁盘信息：squidclient -p 80 mgr:diskd
更新缓存：squidclient -p 80 -m PURGE http://www.xxx.com/xxx.php

补充：

如果squidclient访问被拒绝；则因为没有设置此客户端IP可以管理进程。
配置
http_access allow localhsot(or YourIP) manager
http_access deny manager

定时滚动日志

例如，如下任务接口在每天的早上4点滚动日志：
0 4 * * * /usr/local/squid/sbin/squid -k rotate
首先，它关闭当前打开的日志文件。然后，通过在文件名后加数字扩展名，它重命名cache.log,store.log,和access.log。

日志记录关键词

TCP_MEM_HIT 内存缓存命中
TCP_HIT 硬盘缓存命中
TCP_MISS 未命中
