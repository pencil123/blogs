FPM 全局选项
admin 2015/02/26 2018-01-11

pid string
PID文件的位置。默认为空

error_log string
错误日志的位置。

syslog.facility = daemon
指定记录日志的程序，默认值：守护进程

syslog.ident = php-fpm
syslog_ident is prepended to every message

log_level = notice
日志级别

emergency_restart_threshold int
如果子进程在 emergency_restart_interval 设定的时间内收到该参数设定次数的 SIGSEGV 或者 SIGBUS退出信息号，则FPM会重新启动。
0 表示“关闭该功能”。默认值：0（关闭）。

emergency_restart_interval mixed
emergency_restart_interval 用于设定平滑重启的间隔时间。这么做有助于解决加速器中共享内存的使用问题。
可用单位：s（秒），m（分），h（小时）或者 d（天）。默认单位：s（秒）。默认值：0（关闭）。

process_control_timeout mixed
设置子进程接受主进程复用信号的超时时间。
可用单位：s（秒），m（分），h（小时）或者 d（天）。默认单位：s（秒）。默认值：0（关闭）。

process.max = 128
动态PM情况下，最大的fpm进程数。
The maximum number of processes FPM will fork. This has been design to control the global number of processes when using dynamic PM within a lot of pools. Use it with caution.
Default value: 0.表示无限制

process.priority = -19
master进程优先级
Specify the nice(2) priority to apply to the master process (only if set). The value can vary from -19 (highest priority) to 20 (lower priority).
Default value: not set.

daemonize = yes
设置fpm后台运行

rlimit_files = 1024
Set open file descriptor rlimit for the master process.

rlimit_core = 0
Set max core size rlimit for the master process. Default value: 0.

events.mechanism = epoll
指定FPM将使用的事件机制。Specify the event mechanism FPM will use

systemd_interval = 10
当使用systemd集成构建FPM时，请指定在系统健康状况报告通知之间的时间间隔（秒）
