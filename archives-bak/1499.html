MongoDB 命令调优
admin 2016/11/17 2018-03-29

explain() 执行计划

提供了查询信息，使用索引及查询统计等。有利于我们对索引的优化。
db.collection.find({age:10}).explain()

indexOnly: 字段为 true ，表示使用了索引。
cursor：因为这个查询使用了索引，MongoDB 中索引存储在B树结构中，所以这是也使用了 BtreeCursor 类型的游标。如果没有使用索引，游标的类型是 BasicCursor。
n：当前查询返回的文档数量。
nscanned/nscannedObjects：表明当前这次查询一共扫描了集合中多少个文档，我们的目的是，让这个数值和返回文档的数量越接近越好。
millis：当前查询所需时间，毫秒数。
indexBounds：当前查询具体使用的索引。

profile 优化器

3个级别

  • 0： 不开启profiling
  • 1： 将处理时间超过某个阈值(默认100ms)的请求都记录到DB下的system.profile集合;类似于mysql的slowlog
  • 2： 将所有的请求都记录到DB下的system.profile集合

管理
1、在启动mongodb时，添加到启动参数中-profile=1;
2、在客户端设置；设置db.setProfilingLevel(1,time_slowlog);查看db.getProfilingLevel()

查看慢日志

db.system.profile.find({millis:{$gt:5}})

字段说明

  • ts： 该命令在何时执行
  • info: 本命令的详细信息
  • reslen: 返回结果集的大小
  • nscanned: 本次查询扫描的记录数
  • nreturned: 本次查询实际返回的结果集
  • millis: 该命令执行耗时，以毫秒记

优化步骤

  • 创建索引
  • 限制查询的数量
  • 限制查询的字段
  • Capped Collection
  • hint 强制使用索引
  • profiling
