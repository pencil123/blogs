套接字分析工具 SS
admin 2016/08/09 2018-04-28

ss（socket statistics）命令用来显示处于活动状态的套接字信息。
ss命令可以用来获取socket统计信息，它可以显示和netstat类似的内容。但ss的优势在于它能够显示更多更详细的有关TCP和连接状态的信息，而且比netstat更快速更高效。当服务器的socket连接数量变得非常大时，无论是使用netstat命令还是直接cat /proc/net/tcp，执行速度都会很慢。
ss快的秘诀在于，它利用到了TCP协议栈中tcp_diag。tcp_diag是一个用于分析统计的模块，可以获得Linux 内核中第一手的信息，这就确保了ss的快捷高效。当然，如果你的系统中没有tcp_diag，ss也可以正常运行，只是效率会变得稍慢。

SS 命令使用

Usage: ss [ OPTIONS ]
ss [ OPTIONS ] [ FILTER ]

选项参数

-n, –numeric 不解析服务名称
-r, –resolve 解析解析主机名
-a, –all 显示所有套接字
-l, –listening 显示监听套接字
-o, –options 显示timer信息
-e, –extended 显示详细的套接字信息
-m, –memory 显示套接字的内存使用情况
-p, –processes 显示套接字使用的进程
-i, –info 显示内部TCP信息
-s, –summary 显示套接字使用摘要

网络协议

-4, –ipv4 只显示IPV4
-6, –ipv6 只显示IPV6
-0, –packet display PACKET sockets
-t, –tcp display only TCP sockets
-u, –udp display only UDP sockets
-d, –dccp display only DCCP sockets
-w, –raw display only RAW sockets
-x, –unix display only Unix domain sockets
-f, –family=FAMILY display sockets of type FAMILY

Qurey和Filter

-A, –query=QUERY, –socket=QUERY QUERY := {all|inet|tcp|udp|raw|unix|packet|netlink}[,QUERY]
-D, –diag=FILE Dump raw information about TCP sockets to FILE
-F, –filter=FILE read filter information from FILE
FILTER := [ state TCP-STATE ] [ EXPRESSION ]

状态过滤参数

listen is used for service listening a port or socket
established used for all ready created connection
syn-sent used for session creation is started for the TCP connection but not competed
The state can be either of the following
1. established
2. syn-sent
3. syn-recv
4. fin-wait-1
5. fin-wait-2
6. time-wait
7. closed
8. close-wait
9. last-ack
10. closing
11. all – All of the above states
12. connected – All the states except for listen and closed
13. synchronized – All the connected states except for syn-sent
14. bucket – Show states, which are maintained as minisockets, i.e. time-wait and syn-recv.
15. big – Opposite to bucket state.

连接过滤参数

dst src 远程连接IP和本地连接IP
ss dport gt :PORT 远程端口和一个数比较；
ss sport lt :PORT 本地端口和一个数比较。
端口范围设置

<= or le 小于或等于端口号
>= or ge 大于或等于端口号
== or eq 等于端口号
!= or ne 不等于端口号
< or gt 小于端口号
> or lt 大于端口号

输出说明

State：套接字状态；

Recv-Q/Send-Q 字段

在state为Listen和非Listen时，Recv-Q/Send-Q两个参数的值含义不同。
Listen状态
Recv-Q: 当前等待服务器调用accept完成三次握手的listen backlog数据。
Recv-Q：表示listen backlog(半连接)可以使用的最大值。
非Listen状态
Recv-Q：接受队列，表示收到的数据已经在本地接受缓存，但是还未被进程取走。
Send-Q：发送队列，表示数据已经发送，但还未收到对方确认信息。
这两个值通常应该为0，如果不为0可能是有问题的。packets在两个队列里都不应该有堆积状态。可接受短暂的非0情况。

示例

1、无需解析，服务名称TCP
ss -nt
2、显示监听套接字
ss -ltn
3、显示统计信息
ss -s
4、定时器，显示连接持续时间
ss -tn -o
5、显示ipv4
ss -tl4
6、显示套接字使用进程
ss -t -p
7、状态过滤器
ss -t state established
8、IP过滤器
ss dst 192.168.0.1
ss src 192.168.0.1
9、端口过滤
ss '( dport = :22 or sport = :22)'
