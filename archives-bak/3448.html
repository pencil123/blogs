Bacula 控制器配置文件说明1
admin 2018/02/06 2018-03-26

Director 配置

Configuring the Director(bacula-dir.conf file)配置中最为复杂的文件。包含了各种资源的设定。

资源类型

作业-Job，作业定义-JobDefs，客户端-Client，存储-Storage，编目-Catalog，计划-Schedule，文件集-FileSet，池-Pool，控制器-Director或者消息-Messages。
一切都是围绕着作业定义的，并以某种方式与作业关联起来。

控制器资源 定义控制器名称和验证密码。配置文件只能有一个Director
作业资源 定义备份/还原作业，并将客户端，文件集，计划资源连接在一起，用于定义一个作业。
作业定义资源 为作业资源设置默认选项。
计划资源 定义Bacula内部调度程序何时自动运行Job。每个工作只会引用一个。
文件集资源 定义要备份的文件集，和客户端结合使用。每个作业只能引用一个文件集。
客户端资源 定义要备份的客户端。每个工作只会引用一个客户端。
存储资源 定义应在哪个物理设备上安装卷。您可能有一个或多个存储定义。
池资源 定义可用于特定作业的卷池。大多数人使用一个默认的池。
但是，如果您拥有大量的客户端或卷，则可能需要拥有多个池。
池允许您限制作业（或客户端）仅使用特定的一组卷。
编目资源 定义在哪个数据库中保存文件列表以及备份卷的名称。大多数人只使用一个目录
消息资源 定义错误和信息消息的发送或记录。

控制器资源 Director

Name required
Password 用于校验Console连接;required
DirPort 端口号
QueryFile 解析Console命令对应的SQL语句;required
WorkingDirectory 存储状态文件的目录;required
Maximum Concurrent Jobs 同时运行的最大任务数量
Messages 定义控制器的消息传递位置|方式

作业资源 Job

Name 作业的名称；required
Type Backup|Restore|Verify|Admin
Level 备份作业的等级：Full,Incremental，Differential
JobDefs 继承作业定义资源的默认值
Client 客户端资源名称
FileSet 文件集资源名称
Storage 存储资源名称
Schedule 计划资源名称
Message 消息资源名称
Poll 池资源名称
Priority 优先级
Write Bootstrap Bacula的会写一个引导每个备份作业运行文件。
该指令仅适用于备份作业。
如果备份作业是完整保存，则在写入引导记录之前，Bacula将清除指定文件的所有当前内容。
如果作业是增量或差异保存，Bacula会将当前的引导记录附加到文件末尾。

补充：还原作业需要定义FileSet，但是用不着。

作业定义资源 JobDefs

JobDefs资源允许可以出现在Job资源中的所有相同的指令。但是，JobDefs资源不会创建作业，而是可以在作业中引用它，为该作业提供默认值。这允许您简洁地定义几个几乎相同的作业，每个作业都引用一个包含默认值的JobDefs资源。每个作业只需要提及默认的更改。
