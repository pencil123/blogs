logstash command
admin 2017/11/08 2017-11-08

管理常用命令：

#检查配置文件
logstash -f config-pipe.conf -t

#启动服务：
logstash -f config-pipe.conf

#停止服务：
kill -TERM {logstash_pid}

#重新加载配置文件：
kill -HUP {logstash_pid}
logstash -r

Usage:
bin/logstash [OPTIONS]

Options:

-n, –node.name NAME 节点名称
-f, –path.config CONFIG_PATH 管道配置文件或目录；
-e, –config.string CONFIG_STRING 给定字符串配置
–modules MODULES Load Logstash modules.
-M, –modules.variable MODULES_VARIABLE 加载模块模板的变量
–setup 将索引模板加载到Elasticsearch中，并在运行模块时将搜索，索引模式，可视化和仪表板保存到Kibana中。(default: false)
-w, –pipeline.workers COUNT 管道工作线程的数量
-b, –pipeline.batch.size SIZE 该选项定义了在尝试执行过滤器和输出之前，单个工作线程将从输入中收集的事件的最大数量。
-u, –pipeline.batch.delay DELAY_IN_MS 在创建管道批次时，轮询下一个事件需要多长时间
–pipeline.unsafe_shutdown 强制logstash在关机过程中退出，即使内存中仍存在飞行事件。 默认情况下，logstash将拒绝退出，直到所有收到的事件都被推送到输出。(default: false)
–path.data PATH
-p, –path.plugins PATH
-l, –path.logs PATH
–log.level LEVEL
–config.debug 调试
-i, –interactive SHELL Drop to shell instead of running as normal. Valid shells are “irb” and “pry”
-V, –version
-t, –config.test_and_exit Check configuration for valid syntax and then exit.
-r, –config.reload.automatic 重新加载配置文件
–config.reload.interval RELOAD_INTERVAL How frequently to poll the configuration location for changes, in seconds. (default: 3)
–http.host HTTP_HOST
–http.port HTTP_PORT
–log.format FORMAT
–path.settings SETTINGS_DIR 配置文件目录
–verbose Set the log level to info.
–debug Set the log level to debug.
–quiet Set the log level to info.
-h, –help print help
