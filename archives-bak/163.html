Nginx 文件缓存
admin 2015/04/25 2018-04-20

有关文件IO类指令:
aio、directio、open_file_cache、open_file_cache_min_uses、open_file_cache_valid和sendfile等；

示例 Example

open_file_cache max=1000 inactive=20s;
open_file_cache_valid 30s;
open_file_cache_min_uses 2;
open_file_cache_errors on;

open_file_cache off;

设置打开文件的缓存
open_file_cache max=N [inactive=time];
Default: off;
Context: http, server, location

配置一个缓存用于存储：
打开的文件描述符；文件大小和修改时间
存在的目录信息
文件检索错误信息；如文件不存在，没有权限
是否缓存文件检索错误信息，使用open_file_cache_errors命令设置

max：sets the maximum number of elements in the cache; on cache overflow the least recently used (LRU) elements are removed;
#缓存的最大数量
inactive：defines a time after which an element is removed from the cache if it has not been accessed during this time; by default, it is 60 seconds;
#缓存的过期时间
off：disables the cache.
#禁用缓存

open_file_cache_errors on | off;

是否缓存文件检索中的错误信息
Default:off;
Context: http, server, location
Enables or disables caching of file lookup errors by open_file_cache.

open_file_cache_min_uses number;

文件缓存最小使用数
Default:1;
Context:http, server, location

Sets the minimum number of file accesses during the period configured by the inactive parameter of the open_file_cache directive, required for a file descriptor to remain open in the cache.
#在open_file_cache指令设置的缓存时间内，文件最小的访问次数；如果超过这个数，则文件描述符会保持缓存中的打开状态。

open_file_cache_valid time;

指定多长时间检查一次缓存信息是否变更
Default:60s;
Context: http, server, location
Sets a time after which open_file_cache elements should be validated.

一种特殊情况

当用户使用新图片，覆盖线上的图片时，如果图片的URI没有更改；那么在刷新图片时，会出现图片更新失败的情况。
即使设置图片不进行缓存(expires 0)依然不生效；浏览器还是会使用本地缓存。
抓包发现，浏览器发现图片过期并发送了请求带有If-Modified-Since，得到的是304且Last-Modified时间没变，也就是说nginx说此图片没有更改，浏览器就用本地缓存了。

原因就是nginx文件缓存的设置：

open_file_cache max=102400 inactive=20s;
open_file_cache_valid 30s;
open_file_cache_min_uses 1;
文件信息将会给缓存20秒；
每30秒才会检查一次缓存信息是否更改；
如果20s内使用过一次文件，文件描述符将持续缓存。
