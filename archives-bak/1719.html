Nginx Location 指令说明
admin 2015/06/24 2018-04-19

location[ = | ~ | ~* | ^~ ] uri { ... }
location @name { ... }
Default: —
Context: server, location
该指令允许对不同的URI进行不同的配置，既可以使用字符串，也可以使用正则表达式。

匹配类型

正则表达式：
~ 表示执行一个正则匹配，区分大小写
~* 表示执行一个正则匹配，不区分大小写
!~ 区分大小写不匹配的正则
!~* 不区分大小写不匹配的正则

前缀字符串定义：
= 进行普通字符精确完全匹配。如果匹配成功，则停止继续匹配
^~ 表示普通字符匹配。使用前缀匹配。如果匹配成功，则不再匹配正则表达式
KONG 前缀字符串匹配；URI从第一个字符开始匹配；但是正则表达式及较长的字符串将会被优先匹配

特殊：
@ 是一个命名标记，这种location不会用于正常的请求；通常用于处理内部的重定向时。例如 error_page, try_files

匹配过程

  • 首先检查使用字符串定义的Location。其中，选择并记住具有最长匹配的字符串定义的location。
  • 然后检查正则表达式定义的location，按照它们在配置文件中的顺序依次检查。
  • 正则表达式搜索在第一个匹配的Location终止，并且使用相应的配置。
  • 如果与正则表达式不匹配，则使用之前记住的前缀Location配置。

案例：

location = / {
# 只匹配"/".
[ configuration A ]
}
location / {
# 匹配任何请求，因为所有请求都是以"/"开始
# 但是更长字符匹配或者正则表达式匹配会优先匹配
[ configuration B ]
}
location ^~ /images/ {
# 匹配任何以 /images/ 开始的请求，匹配成功后，不会去匹配正则表达式
[ configuration C ]
}
location ~* .(gif|jpg|jpeg)$ {
# 匹配以 gif, jpg, or jpeg结尾的请求.
# 但是所有 /images/ 目录的请求将由 [Configuration C]处理.
[ configuration D ]
}
/ -> 符合configuration A
/documents/document.html -> 符合configuration B
/images/1.gif -> 符合configuration C
/documents/1.jpg ->符合 configuration D
