python gandi 域名管理接口使用
admin 2016/08/11 2016-10-15

这里主要记录一下使用gandi的api域名接口的经历；不定期更新。
其中比较简单的就不详细说明了。
域名接口有好几类，包括：域名注册，域名各种记录的管理……
最近写的一个脚本只要是修改www二级域名的A记录的。loveu.lv是我在gandi上注册的域名。
gandi上对域名的管理方式有些特别；它维护这若干个”域名区域档“，区域档下又存在不同的版本。当一个域名和一个区域档相关联后，域名的各种记录信息，就是记录在区域档中active（生效）版本中的。
1、首先获取域名是在哪个版本名下的；这里直接跨越了区域档的id；
#Get the work domain id
domain_infos = api.domain.info(apikey,’loveu.lv’)
zone_work_id = domain_infos[‘zone_id’]

2、获取目前区域档下active的版本num
#Get the work domain record version id
zones = api.domain.zone.list(apikey)
print zones
for zone_id in zones:
if zone_id[‘id’] == zone_work_id:
zone_work_tag = zone_id
zone_record_version = zone_work_tag[‘version’]

3、不能更改active状态的版本；所以要更改一个没有使用的版本，然后将版本设置为active状态；
我的这个是使用了 7 和8 版本。
#update the other record version
if zone_record_version == 8:
update_version = 7
else:
update_version = 8

4、获取要更新的record记录
#get the record version www domain id
record_infos = api.domain.zone.record.list(apikey,zone_work_id,update_version)
for record_info in record_infos:
if record_info[‘name’] == ‘www’:
update_record_id = record_info[‘id’]
print update_record_id
5、更新记录
#execute the domain record update
api.domain.zone.record.update(apikey,zone_work_id,update_version,{‘id’:update_record_id},{‘ttl’: 840, ‘type’: ‘A’, ‘name’: ‘www’, ‘value’: ‘127.0.0.3’})
6、设置新的版本为生效版本
api.domain.zone.version.set(apikey,zone_work_id,update_record_id)
