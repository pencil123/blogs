MySQL 多表操作
admin 2015/10/20 2018-04-11

多表查询

关系型数据库设计表结构时，要求符合范式规范。在搜索数据时，常会涉及到跨表查询。

交叉查询（连接）

笛卡尔乘积 cross join:将一张表中的任意一行记录和其他表的任意一行记录结合。
select * from t1,t2,t3;
select * from t1 cross join t2 cross join t3;

内连接:inner join

1、等值连接：select * from t1,t2 where t1.col1 = t2.col2;
2、不等连接：select * from t1,t2 where t1.col1 > t2.col2;
3、自连接：select * from students as s1 inner join students as s2 on s1.col1 = s2.col1;
4、自然连接

外连接

左外连接

以左边的表为基准 left join
select * from t1 left outer join t2 on t1.id=t2.id;

右外连接

以右边的表为基准 right join
select * from t1 right outer join t2 on t1.id=t2.id;

子查询

在查询语句中嵌套查询语句；基于某语句的查询结果再次进行的查询

where子句中的子查询

1、用于比较表达式中的子查询，子查询仅能返回单个值：
select name,age from students where age > (select avg(age) from students);
2、用于IN中的子查询，子查询应该单键查询并返回一个或者多个值从而构成列表。
select name,age from students where age in (select age from teachers);
3、用于exists

用于from子句中的子查询

select table_alias.col… from (select clause) as table_alias where clause;
示例：
select s.aage,s.classid from (select avg(age) as aage,classid from students where classid is not null group by classid) as s where s.aage >10;

联合查询

UNION 把多个查询语句的结果集中显示在一起
select name,age from students union select name,age from teachers;

多表更新

更新表A中的A1为B1、A2为B2；条件为ID相同

UPDATE A, B SET A1 = B1, A2 = B2 WHERE A.ID = B.ID

更新表A 中num字段为A.id在表B的id字段中count的值

update user a set a.posts=
(select b.counts from
(select authorid,count(*) counts from post group by authorid) b where a.uid=b.authorid)
两层嵌套；
先将B字段中authorid,count(*) 查询出来；做为表B；
然后a.num = b.counts

插入数据

insert into test(name,num) select from name,count(*) group by name;

表拷贝

在MySQL管理过程中，使用过两个表拷贝的方式。踩了一次坑，希望大家引以为戒。

使用as语句

create table test as select * from test_old;这里的as 关键词可以省略掉。
这种方式虽然将数据表创建和数据拷贝简化到一个命令中，但是其中却有很大的坑。
新的数据表的表结构是根据test_old表中的数据创建的；而没有拷贝表字段。这样的话test_old表中的索引和字段中的修饰符(如auto_increment)都没有拷贝到。

使用like语句分步拷贝

create table test like test_old
使用like语句，拷贝表结构；会包括索引和字段的修饰符
insert into test select * from test_old;
