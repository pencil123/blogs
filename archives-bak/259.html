es 仓库|快照
admin 2016/06/10 2016-08-18

Es用来处理大数据的分析非常方便和高效。非常大量的数据自然涉及到备份和迁移困难的问题。Es官方提供了数据建立快照的接口，方便对数据的备份和恢复。
Es通过快照实现数据的备份处理，建立快照之前需要创建好仓库；然后就快照存储在仓库中。可以针对部分索引创建快照，也可以对所有的索引创建快照。
创建仓库
a首先需要在所有es节点中elasticsearch.yml文件中定义好仓库的路径。
path.repo: [“/mount/back”]
并且这个目录是共享的目录，所有的节点都能对这个目录进行读写。
b 然后就需要在es执行创建仓库的指令了。
仓库的类型支持Shared filesystem, Amazon S3, HDFS和Azure Cloud
curl -XPUT http://127.0.0.1:9200/_snapshot/my_backup
{
“type”: “fs”,
“settings”: {“location”: “/data”},
“max_snapshot_bytes_per_sec” : “50mb”,
“max_restore_bytes_per_sec” : “50mb”
}
说明：创建名为my_backup的仓库；
仓库目录为：/data
限定最大快照生成速度为50mb
限定最大恢复速度问题50mb

快照
建立所有索引的快照
curl -XPUT http://127.0.0.1:9200/_snapshot/my_backup/snapshot_1
建立部分索引的快照 注意：indices后面需要有一个空格
curl -XPUT http://127.0.0.1:9200/_snapshot/my_backup/snapshot_1
{
“indices”: “index_1,index_2”
}

或者
curl -XPUT “http://127.0.0.1:9200/_snapshot/my_backup/snapshot_${BackNum}” -d ‘{“indices”:”logstash-‘${IndexDay}'”}’
撤销创建过程
如果备份正在后台进行，也可以直接删除来取消此次备份。

查看快照信息
curl http://127.0.0.1:9200/_snapshot/my_backup/snapshot_1
更详细的信息可以使用
curl http://127.0.0.1:9200/_snapshot/my_backup/snapshot_1/_status
恢复快照
恢复快照中全部索引：
POST http://127.0.0.1:9200/_snapshot/my_backup/snapshot_1/_restore
这个api还有额外的参数：恢复指定的索引
POST http://127.0.0.1:9200/_snapshot/my_backup/snapshot_1/_restore
{
“indices”: “index_1”,
“rename_pattern”: “index_(.+)”,
“rename_replacement”: “restored_index_$1”
}

删除快照
curl -XDELETE http://127.0.0.1:9200/_snapshot/my_backup/snapshot_1
