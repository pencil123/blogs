PHP IO编程之同步阻塞
uptoyou 2018/05/11 2018-05-15

阻塞与非阻塞 阻塞与非阻塞是针对服务端来说的 阻塞IO是指调用结果返回之前，当前进程/线程会被挂起； 非阻塞是指在不能立刻得到结果之前，该函数不会阻塞当前进程/线程，而会立刻返回。 同步与异步 同步与异步是针对客户端来说的 同步是发出一个请求后，在没有得到结果之前，该请求就不返回； 异步是发送一个请求后，请求者不能立刻得到结果，等到请求处理完成后，通过状态、通知和回调来通知请求者。
多路复用：能同时处理多个连接模型称为多路复用，比较常见的有select、poll、epoll、kqueue等IO模型
同步阻塞IO模型
$serv = stream_socket_server(“tcp://0.0.0.0:8000”, $errno, $errstr) or die(“create server failed”);

while (1) {
$conn = stream_socket_accept($serv);

$request = fread($conn, 1024);

fwrite($conn, $request);
fclose($conn);
}
stream_socket_accept会造成IO阻塞，当有连接进来时才会响应 fread也会造成阻塞，等待接收数据后才会响应
多进程同步阻塞IO模型
$serv = stream_socket_server(“tcp://0.0.0.0:8000”, $errno, $errstr) or die(“create server failed”);
while (1) {
$conn = stream_socket_accept($serv);
if (pcntl_fork() == 0) {
$request = fread($conn, 1024);
fwrite($conn, $request);
fclose($conn);
}
}

每进来一个连接，就会生成一个进程，但是这种模式进程/线程的创建和销毁会造成系统资源消耗
可复用多进程同步阻塞IO模型
$serv = stream_socket_server(“tcp://0.0.0.0:8000”, $errno, $errstr) or die(“create server failed”);

for ($i=0; $i < 10; $i++) {
if (pcntl_fork() == 0) {
while (1) {
$conn = stream_socket_accept($serv);
if ($conn) {
continue;
}
$request = fread($conn, 1024);
fwrite($conn, "hello world");
fclose($conn);
}

exit(0);
}
}
程序执行时会生成10个子进程，当有连接进来时，会使用其中一个子进程处理连接，当连接关闭时，子进程会被释放，重新进入stream_socket_accept阻塞，继续等待新的连接 这种模式相比前一种模式，没有反复创建销毁进程造成的资源消耗 但是这种模式也有缺陷，因为一个连接会占用一个进程，系统创建进程有限，而且启动大量进程会带来额外的进程调度消耗，进程越多，消耗越多；如果是类似聊天程序（IM），一台服务器要维持上万到上百万的连接（C10K）就显得很吃力
