管道配置文件
admin 2017/11/08 2017-11-08

配置文件的结构：

首先：定义事件处理管道（input\filter\output）;可以在管道中添加多个插件支持。
然后：插件的配置由插件和插件参数组成一个设置块。

数值类型：
数组：users => [ {id => 1, name => bob}, {id => 2, name => jane} ]
列表：path => [ “/var/log/messages”, “/var/log/*.log” ]
布尔：ssl_enable => true
字节：my_bytes => “100kib” # 102400 bytes
解码器：codec => “json” #A list of available codecs can be found at the Codec Plugins page.https://www.elastic.co/guide/en/logstash/current/codec-plugins.html
hash:match => {
"field1" => "value1"
"field2" => "value2"
...
}
number:port => 33
password:my_password => “password”
uri: my_uri => “http://foo:bar@example.net”
path:my_path => “/tmp/logstash”
string:
转移符：\t:tab (ASCII 9)…
注释：#Comments

Accessing Event Data and Fields in the Configuration
All events have properties. For example, an apache access log would have things like status code (200, 404), request path (“/”, “index.html”), HTTP verb (GET, POST), client IP address, etc. Logstash calls these properties “fields.”

条件：
equality: ==, !=, <, >, <=, >=
regexp: =~, !~ (checks a pattern on the right against a string value on the left)
inclusion: in, not in
The supported boolean operators are:and, or, nand, xor
The supported unary operators are:!

使用环境变量
声明：
export TCP_PORT=12345
使用：
input {
tcp {
port => "${TCP_PORT}"
}
}
