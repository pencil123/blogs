Nginx 用户认证和访问控制403
admin 2015/05/21 2018-05-10

访问控制

模块：ngx_http_access_module
设置整个nginx服务、单个虚拟主机、某个主机的某个路径 允许|拒绝访问的IP或者ip端；
如果拒绝访问返回403状态码

allow允许

语法: allow address | CIDR | unix: | all;
默认值: —
配置段: http, server, location, limit_except
允许某个ip或者一个ip段访问.如果指定unix:,那将允许socket的访问.

deny拒绝

语法: deny address | CIDR | unix: | all;
默认值: —
配置段: http, server, location, limit_except
禁止某个ip或者一个ip段访问.如果指定unix:,那将禁止socket的访问

403 forbidden 说明

Nginx访问出现”403 拒绝访问”状态的三种原因

1、缺少索引文件

index index.html index.php;
root /data/www;
如果目录中没有index.html 和index.php文件，则会403错误。

2、代码文件权限问题

Nginx服务运行用户，没有权限访问代码目录。

3、Nginx配置403拒绝访问

nginx 可以设置访问控制allow 和deny;
判断用户是否拒绝访问的维度也有很多；如：user_agent，request_uri，access_ip等

用户认证

模块:ngx_http_auth_basic_module
Nginx默认安装ngx_http_auth_basic_module模块；
如果要禁用次模块，编译按照中使用–without-http_auth_basic_module 。

auth_basic 对话框名称

语法: auth_basic string | off;
默认值: auth_basic off;
配置段: http, server, location, limit_except
默认表示不开启认证，后面如果跟上字符，这些字符会在弹窗中显示。

auth_basic_user_file 用户密码文件

语法: auth_basic_user_file file;
默认值: —
配置段: http, server, location, limit_except
用户密码文件；使用htpasswd或者perl创建密码。

配置案例：

location /
{
auth_basic "nginx basic http test for ttlsa.com";
auth_basic_user_file conf/htpasswd;
autoindex on;
}

Htpasswd 工具

由Apache软件提供的工具。htpasswd用于创建和更新用于存储HTTP用户基本身份验证用户名和密码的flat-files。

htpasswd选项参数

htpasswd [ -c ] [ -i ] [ -m | -B | -d | -s | -p ] [ -C cost ] [ -D ] [ -v ] passwdfile username
htpasswd -b [ -c ] [ -m | -B | -d | -s | -p ] [ -C cost ] [ -D ] [ -v ] passwdfile username password
htpasswd -n [ -i ] [ -m | -B | -d | -s | -p ] [ -C cost ] username
htpasswd -nb [ -m | -B | -d | -s | -p ] [ -C cost ] username password

常用参数说明

-c 创建一个加密文件
-n 在标准输出上显示结果而不是更新文件
-m 使用MD5加密密码。这是默认值
-d 使用crypt()加密密码
-p 使用明文密码
-s 使用SHA加密密码
-b 从命令行获取密码而不是提示输入密码。应该非常小心地使用此选项，因为密码在命令行中清晰可见。
-D 删除用户。如果用户名存在于指定的htpasswd文件中，它将被删除。

使用示例

1、创建文件并创建用户
htpasswd -bc .passwd username pass
创建.passwd文件，用户名username ，密码：pass，默认采用MD5加密方式

2、原文件添加用户
htpasswd -b .passwd onlyzq pass
去掉c选项,即添加用户

3、取得加密密码
htpasswd -nb username pass
在终端中输出username用户和pass加密后的密码

4、删除用户
htpasswd -D .passwd username
