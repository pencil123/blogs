Paths and Directories

include_path string
Specifies a list of directories where the require, include, fopen(), file(), readfile() and file_get_contents() functions look for files.
指定require，include，fopen（），file（），readfile（）和file_get_contents（）函数查找文件的目录列表。

open_basedir string
Limit the files that can be accessed by PHP to the specified directory-tree, including the file itself.
将可以由PHP访问的文件限制在指定的目录树中，包括文件本身。

doc_root string
PHP’s “root directory” on the server. Only used if non-empty. If PHP is configured with safe mode, no files outside this directory are served. If PHP was not compiled with FORCE_REDIRECT, you should set doc_root if you are running PHP as a CGI under any web server (other than IIS). The alternative is to use the cgi.force_redirect configuration below.
PHP的“根目录”在服务器上。 仅在非空时才使用。 如果PHP配置为安全模式，则不会在该目录之外提供文件。 如果PHP没有使用FORCE_REDIRECT编译，那么如果您在任何Web服务器（IIS除外）上运行PHP作为CGI，应该设置doc_root。 另一种方法是使用下面的cgi.force_redirect配置。

user_dir string
The base name of the directory used on a user’s home directory for PHP files, for example public_html .
用户主目录中用于PHP文件的目录的基本名称，例如public_html。

extension_dir string
In what directory PHP should look for dynamically loadable extensions. See also: enable_dl, and dl().
在什么目录PHP应该寻找动态可加载的扩展。 另请参见：enable_dl和dl（）。

extension string
PHP启动时加载的动态加载扩展。
; 要放置临时文件的目录。
; 默认为系统默认值（请参见sys_get_temp_dir）
; sys_temp_dir =“/ tmp”

enable_dl = Off
是否启用dl（）函数。 在多线程服务器（如IIS或Zeus）中，dl（）函数无法正常工作，并在其上自动禁用。

cgi.force_redirect boolean
cgi.force_redirect是提供在大多数Web服务器下运行PHP作为CGI的安全性所必需的。 左未定义，默认情况下，PHP会打开此选项。 您可以自行将其关闭。

cgi.nph boolean
如果cgi.nph被启用，它将强制cgi始终发送状态：200每个请求。

cgi.redirect_status_env string
如果cgi.force_redirect已打开，并且您没有在Apache或Netscape（iPlanet）网络服务器上运行，则可能需要设置一个环境变量名称，PHP将会查找可以继续执行的名称。

cgi.fix_pathinfo boolean
为CGI提供真正的PATH_INFO / PATH_TRANSLATED支持。
PHP的以前的行为是将PATH_TRANSLATED设置为SCRIPT_FILENAME，并且不要弄清PATH_INFO是什么。
有关PATH_INFO的更多信息，请参阅CGI规范。 将其设置为1将导致PHP CGI修复其路径以符合规范。
设置为零会导致PHP的行为像以前一样。 默认情况下打开。 您应该修复脚本以使用SCRIPT_FILENAME而不是PATH_TRANSLATED。

fastcgi.impersonate string
IIS下的FastCGI（基于WINNT的操作系统）支持模拟主叫客户端的安全令牌的能力。 这允许IIS定义请求在其下运行的安全上下文。 Apache下的mod_fastcgi目前不支持此功能（03/17/2002）如果在IIS下运行，请设置为1。 默认为零。

fastcgi.logging boolean
使用FastCGI时打开SAPI日志记录。 默认是启用日志记录。

cgi.rfc2616_headers int
告诉PHP发送HTTP响应代码时要使用什么类型的标头。
如果设置为0，PHP会发送Apache和其他Web服务器支持的»RFC 3875“Status：”头。
当此选项设置为1时，PHP将发送»符合RFC 2616的标头。
