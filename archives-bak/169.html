keepalive 长连接
admin 2015/04/26 2018-04-20

keepalive_disable none | browser …;
Default:
keepalive_disable msie6;
Disables keep-alive connections with misbehaving（行为不端的） browsers. The browser parameters specify which browsers will be affected. The value msie6 disables keep-alive connections with old versions of MSIE, once a POST request is received. The value safari disables keep-alive connections with Safari and Safari-like browsers on Mac OS X and Mac OS X-like operating systems. The value none enables keep-alive connections with all browsers.
keepalive_disable 可以设定对哪些浏览器不建立长连接；可以用来禁止api接口调用的连接。
Prior to version 1.1.18, the value safari matched all Safari and Safari-like browsers on all operating systems, and keep-alive connections with them were disabled by default.

keepalive_requests number; #连接使用次数
Default:
keepalive_requests 100;
This directive appeared in version 0.8.0.
Sets the maximum number of requests that can be served through one keep-alive connection. After the maximum number of requests are made, the connection is closed.
设置一个keepalive连接处理的最大请求数；当达到这个数值收，keepalive连接将被关闭

keepalive_timeout timeout [header_timeout];#超时时间
Default:
keepalive_timeout 75s;
The first parameter sets a timeout during which a keep-alive client connection will stay open on the server side. The zero value disables keep-alive client connections.
第一个参数：设置一个超时时间；
第二个参数：添加一个header字段
The optional second parameter sets a value in the “Keep-Alive: timeout=time” response header field. Two parameters may differ.
The “Keep-Alive: timeout=time” header field is recognized by Mozilla and Konqueror. MSIE closes keep-alive connections by itself in about 60 seconds.
