elk geoip type is not [geo_point]
admin 2016/06/16 2016-06-16

logstash 根据ip定位地图坐标 字段类型不是[geo_point]类型
I’m attempting to use the geoip filter to populate my logs with geographic coordinate data that I can use to drive the “tile map” visualization in Kibana 4. The problem is that the data retrieved by this filter is mapped as a “double” type instead of a “geo_point” type, which prevents the tile map from recognizing the data as a geographic coordinate data source.

According to the docs:

Starting with version 1.3.0 of Logstash, a [geoip][location] field is created if the GeoIP lookup returns a latitude and longitude. The field is stored in GeoJSON format. Additionally, the default Elasticsearch template provided with the elasticsearch output maps the [geoip][location] field to a geo_point.

Answer:

The problem is your index name is “access-…”, if you look at the template, it only matches logstash-*.

We are exploring ways to have logstash use the index setting on the elasticsearch output to inform what the index pattern should be in the template, but this isn’t easy. There’s a patch from @Bradox (logstash-plugins/logstash-output-elasticsearch#25) that works towards this, if you want to try it.

For today, there are two possible workarounds:

1) As a workaround, you’ll want to create your own template (which can be based on the default one) that matches your index naming (access-*).

or

2) An alternate workaround would be either use the default index naming (logstash-YYYY.mm.dd) or use logstash- as a prefix, such as logstash-access-%{+YYYY.MM.dd}.
