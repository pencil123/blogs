postfix 基于sasl认证
admin 2017/07/09 2017-10-19

postfix开启sasl认证功能
实现sasl功能的软件有：cyrus-sasl和courier-authlib

cyrus-sasl的安装包：
cyrus-sasl
cyrus-sasl-lib
cyrus-sasl-devel
cyrus-sasl-md5
cyrus-sasl-gssap
cyrus-sasl-plain

saslauthd服务：cyrus-sasl
/etc/rc.d/init.d/saslauthd 管理脚本
/etc/sysconfig/saslauthd 配置文件
/usr/sbin/pluginviewer
/usr/sbin/saslauthd 显示当前主机saslauthd服务所支持的认证机制，默认为pam
/usr/sbin/testsaslauthd 测试sasl认证 -u username -p password
……
MECH：认证机制

使用以下命令验正postfix是否支持cyrus风格的sasl认证，如果您的输出为以下结果，则是支持的：
# /usr/local/postfix/sbin/postconf -a
cyrus
dovecot

postfix配置支持sasl

配置：main.cf文件

1、设置使用哪种sasl认证软件；默认是cyrus
smtpd_sasl_type = cyrus

2、设置smtp服务如何查找认证服务;socket file or server ip:port
smtpd_sasl_path = smptd

3、启动认证
smtpd_sasl_auth_enable = yes
smtpd_banner = Welcome to our $myhostname ESMTP,Warning: Version not Available!
4、认证配置
postfix smtpd服务使用sasl服务是的配置项；
配置项依赖使用的那种smtpd_sasl_type
cyrus服务提供的选项：
noplaintext：禁止使用明文密码
noactive：禁止主动连接
nodictionary：禁止被动连接
noanonymous：禁止匿名身份验证
forward_secrecy：只允许支持前向保密；Only allow methods that support forward secrecy (Dovecot only).
mutual_auth：只允许提供相互认证方法

配置：
smtpd_sasl_security_options = noanonymous

5、smtpd服务本地的sasl认证领域
smtpd_sasl_local_domain = $myhostname

6、兼容非标准smtp认证协议
broken_sasl_auth_clients = yes

7、访问控制：
smtpd_recipient_restrictions=permit_mynetworks,permit_sasl_authenticated,
reject_invalid_hostname,reject_non_fqdn_hostname,reject_unknown_sender_domain,
reject_non_fqdn_sender,reject_non_fqdn_recipient,reject_unknown_recipient_domain,
reject_unauth_pipelining,reject_unauth_destination

指定sasl服务支持的smtpd程序的认证方式
# vim /etc/sasl2/smtpd.conf /usr/lib/sasl2/smtpd.conf
添加如下内容：
pwcheck_method: saslauthd
mech_list: PLAIN LOGIN

更改sasl认证机制为shadow
vim /etc/sysconfig/saslauthd
MECH=shadow

让postfix重新加载配置文件
#/usr/sbin/postfix reload
