Linux 系统密码策略
admin 2018/03/22 2018-03-26

当前的网络环境，黑客行为猖獗地超出一个善良人的想象。
一台在线服务器如果没有限制sshd服务的端口，每天不会少于几百次的暴力破解密码操作。
为了服务器的安全着想，最好要限制可登录IP地址；不然可以限制使用证书登录；再不然就是设置密码机制了。

密码机制

包括几个方面，根据个人情况选择使用。
1、密码质量检测：包括字符设定(数字|大小写|特殊字符)和密码长度
2、密码有效期：一个密码的有效时间
3、防止重复使用旧密码

密码质量检测

pam_cracklib(PAM模块)根据字典单词检测密码
该模块可以嵌入到“应用程序的密码堆栈”中，提供对密码强度的检测功能。
该模块的作用是提示用户输入密码，并根据系统字典和一系列密码规则检查密码的强度。

工作过程

首先提示“输入新密码”，并检测密码的质量；如果密码质量合格，则提示“重新输入密码”，用于检验密码的一致性。
两者都通过后，会将密码提交给后续的模块处理。

密码强度检测过程

1、检测密码是否为字典单词的一部分
2、新密码是否和之前密码相同
3、新密码只是更改了大小写
4、新密码和旧密码相似度（difok）
5、密码是否指令合格（minlen\maxclassrepeat\dcredit\ucredit\lvredit\ocredit）
6、新密码是否为旧密码的前后颠倒
7、检测连续字符
8、太长的单调字符排序
9、包含用户名字
此模块没有设置参数也可以很好的使用到标准UNIX密码加密机制中。以上的检测密码强度的操作，可以更加自己需要选择使用。

配置选择

debug 模块将信息写入到日志中
authtok_type 提供输入密码后的默认操作
retry 用户输入N次错误密码后，断开连接
difok 新密码和旧密码中的字符更改数目
minlen 最小的密码长度；密码每多使用一种类型(以下四种)密码长度可以减一。注意Craklib本身也有长度限制
dcredit 数字（N> = 0）这是新密码中有数字的最大值。 如果您的位数少于或等于N位，则每个数字将计数+1以满足当前的minlen值。
ucredit 大写字母 （N = 1）默认值，这是minlen的推荐值小于10。
lcredit 小写字母 （N <0）这是新密码必须满足的最小小写字母数。
ocredit 其他字符 和以上3个配置相同;注意N=1 包含在N》=0中
minclass 密码中的最少字符类型：上面的4中default:disabled
maxrepeat 拒绝包括多余N个连续相同的字符default:disabled
maxsequence 拒绝包括多余N个的单调字符序列的密码。如‘12345’和‘fedcb’;default:disabled
maxclassrepeat 拒绝包含同一类的N个以上连续字符的密码;default:disabled
reject_username 拒绝新密码是否包含直接或反转形式的用户名
gecoscheck 拒绝新密码包含长度超过3个字符的正向和反向GECOS字段（通常是用户的全名）
enforce_for_root 设置ROOT用户也要遵循设定的规则；default:disabled
use_authtok 该参数用于强制模块不提示用户输入新密码
dictpath 指定cracklib字典的路径

返回值

PAM_SUCCESS 新密码通过所有检查。
PAM_AUTHTOK_ERR 没有输入新密码，无法确定用户名，或新密码未通过强度检查。
PAM_AUTHTOK_RECOVERY_ERR 旧密码不是由以前的模块提供的，或者不是来自用户的请求。
PAM_SERVICE_ERR 发生内部错误。

案例

编辑配置文件/etc/pam.d/system-auth
password required pam_cracklib.so difok=3 minlen=15 dcredit= 2 ocredit=2
password required pam_cracklib.so dcredit=-1 ucredit=-1 ocredit=-1 lcredit=0 minlen=8

密码有效期

配置文件为/etc/login.defs
# Password aging controls:
# PASS_MAX_DAYS Maximum number of days a password may be used.
# PASS_MIN_DAYS Minimum number of days allowed between password changes.
# PASS_MIN_LEN Minimum acceptable password length.
# PASS_WARN_AGE Number of days warning given before a password expires.

命令

chage -l 查看密码有效期信息

防止重复使用旧密码

pam_unix 用于传统密码认证的模块
这是标准的Unix认证模块。它使用来自系统库的标准调用来检索和设置帐户信息以及认证。通常这是从/ etc / passwd和/ etc / shadow文件获得的，如果启用了shadow。

其中的配置参数

remember 每个用户的最后n个密码保存在/ etc / security / opasswd中，以强制密码更改历史记录，并防止用户频繁地在相同密码之间交替。
password required pam_unix.so use_authtok nullok md5
至于pam_unix 模块中其他说明，以后找时间更新
