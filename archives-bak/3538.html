Nginx 页面级缓存配置和管理
admin 2018/02/13 2018-04-20

配置页面级缓存

设置Nginx 页面级缓存可以提供请求的响应速度。

简易设置示例

proxy_cache_path /path/to/cache levels=1:2 keys_zone=my_cache:10m max_size=10g inactive=60m use_temp_path=off;
server {
# ...
location / {
proxy_cache my_cache;
proxy_pass http://my_upstream;
}
}

proxy_cache_path

Context：http
设置缓存信息存储的路径和其他参数。

levels 缓存文件存储的目录层级结构的级数；是否建立多级目录用于存储。
keys_zone 设置一个共享内存区域用于存储缓存键和元数据（如使用计时器）。
max_size 设置硬盘存储缓存大小的上限。
inactive 缓存的内容多长时间没有被访问，内容会被删除。

proxy_cache zone | off;

Context:http, server, location
设定用于缓存的共享内存区域。

proxy_cache_key string;

Default:$scheme$proxy_host$request_uri;
Context:http, server, location
定义缓存信息中的key，生成方式。
proxy_cache_key "$host$request_uri $cookie_user";

proxy_cache_valid [code …] time;

Context:http, server, location
对不同的响应状态设置不同的缓存时间

如果只设定缓存时间
proxy_cache_valid 5m;
那么只缓存200,301和302响应。

也可以使用响应头的字段做缓存时间的判断依据

proxy_cache_bypass string …;

Context:http, server, location
定义不会从缓存中获取响应的条件。如果字符串参数中至少有一个值不为空且不等于“0”，则不会从缓存中获取响应：
proxy_cache_bypass $cookie_nocache $arg_nocache$arg_comment;
proxy_cache_bypass $http_pragma $http_authorization;

proxy_no_cache string …;

语境： http，server，location
定义响应不会保存到缓存的条件。如果字符串参数中至少有一个值不为空且不等于“0”，则不会保存响应：
proxy_no_cache $cookie_nocache $arg_nocache$arg_comment;
proxy_no_cache $http_pragma $http_authorization;

更新缓存

NGINX可以从缓存中删除过期的缓存文件。用以处理核心内容更新后，在nginx上的缓存中没有更新。
缓存在接收到‘清除’的请求，就会清除缓存。‘清除’的请求包含一个自定义的HTTP头或者PURGE的http方法。

proxy_cache_purge string …;

语境： http，server，location
该指令出现在1.5.7版本中
定义将请求视为缓存清除请求的条件。
如果字符串参数中至少有一个值不为空并且不等于“0”，则带有相应缓存键的缓存条目将被删除。通过返回204（无内容）响应来指示成功操作的结果。
如果清除请求的缓存键以星号（“ *”）结尾，则匹配通配符键的所有缓存条目将从缓存中删除。
但是，这些条目将保留在磁盘上，直到它们被删除以进行不活动或由缓存清除程序（1.7.12）处理，或者客户端尝试访问它们。

配置示例：

proxy_cache_path /data/nginx/cache keys_zone=cache_zone:10m;
map $request_method $purge_method {
PURGE 1;
default 0;
}
server {
...
location / {
proxy_pass http://backend;
proxy_cache cache_zone;
proxy_cache_key $uri;
proxy_cache_purge $purge_method;
}
}

限制对清除命令的访问

建议您配置允许发送缓存清除请求的有限数量的IP地址：
geo $purge_allowed {
default 0; # deny from other
10.0.0.1 1; # allow from localhost
192.168.0.0/24 1; # allow from 10.0.0.0/24
}
map $request_method $purge_method {
PURGE $purge_allowed;
default 0;
}
在本例中，NGINX检查请求中是否使用了“PURGE”方法，如果是，则分析客户端IP地址。
如果IP地址列入白名单，则$purge_method设置为$purge_allowed：“1”允许清除，“0”拒绝清除。

从缓存中完全删除文件

要完全删除与星号匹配的缓存文件，您需要激活一个特殊的cache purger进程，该进程将永久遍历所有缓存条目并删除与通配符匹配的条目。
在http区域上，将purger参数添加到proxy_cache_path指令中：
proxy_cache_path /data/nginx/cache levels=1:2 keys_zone=mycache:10m purger=on;

补充

：当缓存的文件很大(如视频)时，需要对大文件进行切片
