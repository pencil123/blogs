wp_query 介绍|属性和方法
admin 2018/02/20 2018-03-26

描述

WP_Query是一个在wp-includes/class-wp-query.php中定义的类，用于处理向WordPress博客发送的复杂的请求。
wp-blog-header.php（或版本2.0中的WP类）提供定义当前请求的$ wp_query对象信息，然后$ wp_query确定它处理的查询类型（可能是类别归档，日期归档，提要 ，或搜索），并提取请求的帖子。
它保留了关于请求的大量信息，可以在稍后的日期提取。

与WP_Query交互

大多数情况下，我们有大量可以使用的函数调用我们需要的数据；而不需要处理或接触内部类和全局变量。

wp_query主要使用场景有两个：

1、确定正在处理的请求类型。

$wp_query的is_*属性保存此类信息；如：$is_single, $is_page, $is_archive等

2、和主循环(The Loop)结合使用

首先, have_posts()通过$wp_query->have_posts()的形式进行调用，以查询是否有文章。
若有，则利用 have_posts() 作为判断条件开始 while 循环。只要有文章要显示就不断地迭代该操作。
在每一次迭代的时候，the_post()函数会建立内部变量$wp_query和全局变量$post（依赖于Template Tags ）,以 $wp_query->the_post() 的形式调用。

属性

$query 由WP类传递给$ wp_query对象的查询字符串
$query_vars 查询变量及其各自值的数组
$queried_object 保存所请求类别\作者\帖子或页面的信息。
$queried_object_id 如果请求是一个类别，作者，永久链接或帖子/页面，则保存相应的ID。
$posts 从数据库中获取请求的帖子
$post_count 显示的帖子数量
$found_posts 找到的帖子总数与当前查询参数匹配
$max_num_pages 总页数。是$ found_posts / $ posts_per_page的结果
$current_post (在The Loop期间可用)当前正在显示的帖子的索引
$post （在The Loop期间可用）当前正在显示的帖子。

补充：$is_single, $is_page, $is_archive, $is_preview, $is_date, $is_year,
$is_month, $is_time, $is_author, $is_category, $is_tag, $is_tax,
$is_search,$is_feed, $is_comment_feed, $is_trackback, $is_home,
$is_404,$is_comments_popup, $is_admin, $is_attachment,
$is_singular, $is_robots, $is_posts_page, $is_paged

方法

方法名称前的＆符号表明它通过引用返回

init()

初始化该对象，将所有属性设置为null，零或false。

parse_query( $query )

获取定义请求的查询字符串，解析并填充除$ posts，$ post_count，$ post和$ current_post之外的所有属性。

parse_query_vars()

重新分析旧的查询字符串。

get( $query_var )

获取一个命名的查询变量。

set( $query_var, $value )

将指定的查询变量设置为特定的值。

&get_posts()

从数据库中提取并返回请求的帖子。还要填充$ posts和$ post_count。

next_post()

前进到$ posts的下一篇文章。
这不会设置全局$ post变量，而只是WP_Query对象的实例变量

the_post()

（在循环中使用）前进到下一篇文章，并设置全局$ post变量。

have_posts()

（在循环中或在循环之前使用）确定是否有剩余的帖子要显示。

rewind_posts()

重置$ current_post和$ post。

&query( $query )

调用parse_query（）和get_posts（）。 返回get_posts（）的结果。

get_queried_object()

如果它尚未设置，则设置$ queried_object并返回它。

get_queried_object_id()

如果它尚未设置，则设置$ queried_object_id并返回它

WP_Query( $query = ” ) (constructor)

如果您提供查询字符串，请使用它调用query()。
