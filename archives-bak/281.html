strace 跟踪进程
admin 2015/09/16 2018-02-12

Strace用来跟踪进程执行时的系统调用和所接收的信号。
Linux系统中进程不能直接访问硬件设备，当进程需要访问硬件设备(比如读取磁盘文件，接收网络数据等等)时，必须由用户态模式切换至内核态模式，通过系统调用访问硬件设备。
strace可以跟踪到一个进程产生的系统调用,包括参数，返回值，执行消耗的时间。

usage

strace [-CdffhiqrtttTvVxxy] [-I n] [-e expr]...
[-a column] [-o file] [-s strsize] [-P path]...
-p pid... / [-D] [-E var=val]... [-u username] PROG [ARGS]

or: strace -c[df] [-I n] [-e expr]... [-O overhead] [-S sortby]
-p pid... / [-D] [-E var=val]... [-u username] PROG [ARGS]

常用参数

-c 统计每一系统调用的所执行的时间,次数和出错的次数等
-d 启用调试输出到stderr
-D 将追踪进程作为独立的子进程运行，not as parent
-f 跟随fork的子进程，-ff – 输出到单独的文件中
-i 系统调用时的打印指令指针
-q 禁止关于附加，分离等的消息
-r 打印出相对时间 -t 绝对时间 -tt — with usecs
-T 显示每一调用所耗的时间.
-v 输出所有的系统调用.一些调用关于环境变量,状态,输入输出等调用由于使用频繁,默认不输出
-x 以十六进制形式输出非标准字符串
-y 打印与文件描述符参数关联的路径
-a column 设置返回值的输出位置.默认 为40
-b execve 分离此系统调用
-o file 将strace的输出写入文件filename
-O overhead 设置系统调用到OVERHEAD usecs的开销
-p pid 追踪指定的进程Id的进程,可以重复
-s strsize 将打印字符串的长度限制为STRSIZE字符（默认值为32）
-S sortby 按系统调用排序：time，calls，name，nothing（默认时间）
-u username 运行命令作为用户名处理setuid、setgid
-E var=val 在 命令的环境中放置var = val
-E var 从命令环境中删除var
-P path 跟踪访问路径

-I 可中断 —

1: 没有信号被阻塞
2: 解码系统调用时，致命信号被阻塞（默认）
3: 致命的信号总是被阻塞（如果’-o FILE PROG’则是默认的）
4: 致命信号和SIGTSTP（^ Z）总是被阻塞

-e expr

指定一个表达式,用来控制如何跟踪.格式：[qualifier=][!]value1[,value2]…
qualifier只能是 trace,abbrev,verbose,raw,signal,read,write其中之一。
value是用来限定的符号或数字.
默认的 qualifier是 trace。
感叹号是否定符号。
例如:-eopen等价于 -e trace=open,表示只跟踪open调用。
而-etrace!=open 表示跟踪除了open以外的其他调用。
有两个特殊的符号 all 和 none。
注意有些shell使用!来执行历史记录里的命令,所以要使用\\。

-e trace=set 只跟踪指定的系统 调用.例如:-e trace=open,close,rean,write表示只跟踪这四个系统调用.默认的为set=all.
-e trace=file 只跟踪有关文件操作的系统调用.
-e trace=process 只跟踪有关进程控制的系统调用.
-e trace=network 跟踪与网络有关的所有系统调用.
-e strace=signal 跟踪所有与系统信号有关的 系统调用
-e trace=ipc 跟踪所有与进程通讯有关的系统调用
-e abbrev=set 设定strace输出的系统调用的结果集.-v 等与 abbrev=none.默认为abbrev=all.
-e raw=set 将指定的系统调用的参数以十六进制显示.
-e signal=set 指定跟踪的系统信号.默认为all.如 signal=!SIGIO(或者signal=!io),表示不跟踪SIGIO信号.
-e read=set 输出从指定文件中读出 的数据.例如: -e read=3,5
-e write=set 输出写入到指定文件中的数据

追踪的结果

每一行都是一条系统调用，等号左边是系统调用的函数名及其参数，右边是该调用的返回值。
strace 显示这些调用的参数并返回符号形式的值。
strace 从内核接收信息，而且不需要以任何特殊的方式来构建内核。
