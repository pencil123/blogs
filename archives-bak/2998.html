Percona 和 DSN 说明
admin 2017/02/22 2018-04-30

Percona是MySQL，MariaDB，MongoDB和RocksDB许多开源软件项目的开发人员。他们表示，他们的所有软件都是“完全开源并且完全免费”的。
该公司的收入来源于开源数据库相关支持，咨询和托管服务。
该公司由Peter Zaitsev和Vadim Tkachenko于2006年创立。其总部位于北卡罗来纳州的罗利，在北美和世界其他地区雇佣了大多数偏远的劳动力。

DSN (Data source name) 数据源名称

Percona Toolkit工具使用DSN来指定如何创建到MySQL服务器的DBD连接。DSN是逗号分隔的key=value部分字符串，例如：
h=host1,P=3306,u=bob

某些工具不使用DSN，但仍使用–host，–user和–password等选项连接到MySQL。 这些工具使用这些选项在幕后自动创建DSN。
其他工具同时使用DSN和上述选项。这些选项为所有未指定选项对应关键部分的DSN提供默认值。例如，如果指定了DSN h = host1和选项–port = 12345，则该工具会自动将P = 12345添加到DSN。

转移字符

DSN通常在命令行中指定，因此必须考虑shell引用和转义。
由于DSN部分之间用逗号分隔，因此DSN值中的文字逗号必须用单个反斜杠（\）进行转义。由于反斜杠是大多数shell的转义字符，因此需要两个反斜杠来传递文字反斜杠。例如，如果用户名是字面意思my,name，那么它必须my\\,name在大多数shell上指定。这适用于DSN和与DSN相关的选项–user。

关键词列表

A ：Default character set for the connection
D ：Default database to use when connecting.
F ：Defaults file for the MySQL client library (my.cnf)
h : MySQL hostname or IP address to connect to.
L : Explicitly enable LOAD DATA LOCAL INFILE.
p : MySQL password to use when connecting.
P : Port number.
S : MySQL socket file.
u : username.

传播

许多工具可让您将值从一个DSN传播到下一个，因此您无需为每个DSN指定所有部分。例如，如果您要为每个DSN指定一个用户名和密码，则可以按以下方式连接到三台主机：
h=host1,u=fred,p=wilma host2 host3
