Bacula 服务安装
admin 2018/02/06 2018-03-26

说明

Bacula 为存储备份相关的信息，需要使用数据库。支持的数据库有MySQL,PostgreSQL,SQLite等。
Bacula 发布的版本分为：源码包，文档包，非核心GUI包，win系统安装程序。

版本号说明

Bacula 的版本号按照major.minor.release 的方式命名。
major:主要版本，一般不会变
minor:次要版本；偶数为发行版，奇数是开发版
release:补丁版本

安装说明

Redhat/Centos 系统编译安装建议参数
#CFLAGS="-g -Wall" ./configure \
--sbindir=/opt/bacula/bin \
--sysconfdir=/opt/bacula/etc \
--enable-smartalloc \
--with-mysql \
--with-working-dir=/opt/bacula/working
--with-pid-dir=/opt/bacula/working \
--with-subsys-dir=/opt/bacula/working
--enable-readline
#make install
补充：
安装过程中如果已经编译过参数(./configure )，想修改编译参数时；使用macke distclean清除编译缓存文件。

单独安装Storage

Bacula 安装过程中会出现二选一的情况：
1、只安装File Daemon 使用-enable-client-only 参数编译
2、否则配置数据库连接
本来安装Storage Daemon 不需要使用数据库；不过还是要添加上，可以使用sqlite轻量级数据库。
安装过程
./configure --prefix=/usr/local/bacula/ --enable-build-stored --disable-build-dird --with-sqlite3
make
make install

目录参数设置

  • -prefix=path
  • -sbindir=binary-path
  • -sysconfdir=config-path
  • -datadir=path

特征参数

-disable-ipv6 禁用IPV6
-enable-smartalloc 内存溢出保护
-enable-BAT 依赖Qt的GUI管理工具
-enable-batch-insert 数据库批量插入数据
-enable-bwx-console 不依赖Qt，支持GTK+,X11的GUI管理工具
-disable-nls 默认使用GNU本地语言支持(NLS)库，可能导致异常。禁用后，使用英语。
-with-sqlite3 数据库
-with-mysql 数据库
-with-postgresql 数据库
-with-openssl 通信加密

密码和用户

-with-dir-password=Password 用于访问Director的密码（通常来自控制台程序）
-with-fd-password=Password 用于访问文件守护进程（通常由Director导入）的密码
-with-sd-password=Password 用于访问存储守护程序（通常由Director导入）的密码
-with-dir-user=User 用于运行Director的Userid；Director必须以超级用户身份启动，但不需要以超级用户身份运行
-with-dir-group=Group 用于运行Director的GroupId
-with-sd-user=User 用于运行存储守护进程的用户标识；存储守护程序必须以root用户身份启动，但不需要以root用户身份运行
-with-sd-group=Group 用于运行存储守护进程的GroupId
-with-fd-user=User 用来运行File守护进程的Userid；File守护程序必须以root身份启动，但不需要以root用户身份运行
-with-fd-group=Group 用来运行File守护进程的GroupId
-with-mon-dir-password=Password 用于从监视器访问目录的密码。如果没有指定，配置会自动创建一个随机密码
-with-mon-fd-password=Password 用于从监视器访问文件守护进程的密码
-with-mon-sd-password=Password 用于从监视器访问存储守护进程的密码
-with-db-name=database-name 指定要在conf文件中使用的数据库名称。默认是bacula
-with-db-user=database-user 指定要在conf文件中使用的数据库用户名。默认是bacula

进程设置

  • -enable-client-only 仅安装File Daemon
  • -enable-build-dird 安装Director Daemon 和Director 管理工具；默认配置，-disable-build-dird禁用
  • -enable-build-stored 安装Storage Daemon；默认配置，-disable-build-stored禁用

进程管理

  • sbin/bacula-dir
  • sbin/bacula-fd
  • sbin/bacula-sd

可以拷贝至/etc/init.d/中设置开机自启动
